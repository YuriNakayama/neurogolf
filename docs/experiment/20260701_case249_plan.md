## 仮説
`task055` は行・列 factor を float16 で作った後、末尾で `Rext16 -> Cast(float32) -> Rext` と `Cext16 -> Cast(float32) -> Cext` を作り、float32 の `M2` と final `Einsum` している。

出力は one-hot 0/1 で、float16 でも完全一致できる可能性が高い。`M2` を float16 化し、final `Einsum` を `Rext16` / `Cext16` へ直結すれば、30x30 級の float32 中間を消せる。

## 手法
1. `/tmp` で `M2` を float16 に変換し、final `Einsum` の入力を `Rext16`, `Cext16` に差し替える。
2. `Rext` / `Cext` Cast node を削除し、graph output dtype を float16 に更新する。
3. `audit_one` で `n_fail=0` と cost 減を確認する。
4. 成功した場合のみ本体 `task055.onnx` に反映し submit する。
5. Public Score が `7181.28` を上回れば採用、上がらなければ破棄する。
