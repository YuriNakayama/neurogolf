# case229 plan: task123 canvas index encoding

## 現状
- 採用済み Public Score floor: 7181.21
- `task123` baseline は `cost=1343` (`params=940`, `memory=403`, `n_fail=0`)。
- graph は上段 5 cell から palette を作り、`canvas_index (30,30)` で 11 色 sequence を 30x30 に展開している。
- `canvas_index` は 900 要素の int32 initializer だが、値は 0..10 の 11 種類だけで、実質 `min(max(row, col), 10)` パターンに見える。

## 対象
- `data/output/onnx/task123.onnx`

## 仮説
`canvas_index` を小さい rank/shape の initializer と `Max`/`Min`/`Pad`/`Gather` などで生成できれば params を大きく削れる可能性がある。一方、生成後の int32 30x30 tensor は memory bytes が大きいため、initializer 900 要素より高くなる可能性がある。

## 検証
- `canvas_index` を 11x11 + Pad に置き換える候補を probe する。
- `row/col` の 30x1/1x30 から `Max`/`Min` で生成する候補を probe する。
- 低 dtype index が ONNX Runtime で許されるか確認する。
- `src.evaluate.scorer.audit_one` で `task123` の全ペア完全一致と cost を確認し、`n_fail=0` かつ cost が下がった候補だけ実 ONNX へ反映する。
