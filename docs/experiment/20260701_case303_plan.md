# 20260701 case303 plan - large ratio prioritization

## Context

The score is logarithmic:

`points = max(1, 25 - ln(max(1, cost)))`

`cost = params + memory_bytes`

Therefore a local improvement is:

`delta = ln(old_cost / new_cost)`

The priority target is not the largest absolute-cost task. It is a task where a
structural replacement can produce a large relative reduction while preserving
`n_fail=0`.

## Policy change

- Reject byte-level cleanup unless it is bundled into a larger same-task rewrite.
- Use `old_cost / new_cost >= 1.02` as the normal implementation threshold.
- Prefer `>= 1.05` when a solver rewrite is required.
- Continue to require exact correctness before submit.
- Do not submit changes based only on local equality aliases without a semantic
  rule proof.
