## 仮説
`task255` には bool mask を float32 へ Cast し、`z` と `MatMul` する箇所が4つある。

```text
left_colsR__precast -> Cast(float32) -> left_colsR
right_colsR__precast -> Cast(float32) -> right_colsR
top_rowsR__precast -> Cast(float32) -> top_rowsR
bottom_rowsR__precast -> Cast(float32) -> bottom_rowsR
```

case249 と同様にこれらを float16 化できれば、mask と MatMul output の memory を削れる可能性がある。ただし `z` は input slice の float32 30x30 なので、MatMul 全入力の型を揃えるには `z` も float16 化する必要があり、追加中間が重い懸念がある。

## 手法
1. `/tmp` で `z -> Cast(float16)` を追加する。
2. 4つの mask Cast を float16 に変更し、4つの MatMul で `z_f16` を使う。
3. `audit_one` で session validity / correctness / cost を確認する。
4. `n_fail=0` かつ cost 減なら本体に反映して submit する。失敗または cost 悪化なら提出しない。
