## 背景
case89 では `task133` の marker branch と correction overlay の全体削除を検証し、全 branch が必要と判明した。

一方、correction overlay は 5 つの group からなる。

- `cc_0..cc_4` を `sg_gate_0..sg_gate_4` で gated update にする
- `Concat(cu_0..cu_4) -> corr_upd`
- `ScatterND(border_grid, corr_idx_c, corr_upd) -> corr_overlay`

case89 は correction overlay 全体の削除であり、個別 group の冗長性は未確認。

## 仮説
一部の correction group は train/test/arc-gen 上で常に不要、または marker branch と重複している可能性がある。該当 group をゼロ化または削除できれば params と memory を減らせる。

## 手法
1. `cc_0..cc_4` の各 group を個別に zero initializer へ置換する候補を作る。
2. `src.evaluate.scorer.audit_one` で `task133` の完全一致を確認する。
3. `n_fail=0` かつ cost 減なら、その group の table 削減を実ファイルに適用して submit する。

## 足切り
- 変更対象は `task133` のみ。
- 全数 `audit_dir` は使わず、`audit_one` のみ行う。
