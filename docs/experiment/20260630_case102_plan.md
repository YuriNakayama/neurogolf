# 20260630 case102 plan - t009 sep mask の false Concat を Pad 化

## 現状
- floor/Public: 7180.95
- 対象: `task009`
- baseline faithful: cost 8410, params 1081, memory 7329, n_fail 0

## 仮説
`task009` は opset 17 で、separator mask 生成時に bool tensor へ all-false initializer を末尾 `Concat` している。

- `v_mask_tail` `(1,1,10,9)` + `false_col` `(1,1,10,1)` -> `v_sep_mask`
- `h_mask_tail` `(1,1,9,10)` + `false_row` `(1,1,1,10)` -> `h_sep_mask`

どちらも suffix 方向の false padding なので、`Pad` へ置換できれば params と intermediate cost を削れる可能性がある。

## 手法
1. `v_sep_mask` と `h_sep_mask` の 2 つの `Concat` を、それぞれ `Pad` に置換する。
2. `false_col` / `false_row` initializer を削除する。
3. `src.evaluate.scorer.audit_one` で `task009` の完全一致を確認する。
4. `n_fail=0` かつ cost 減なら Kaggle submit し、実 Public Score で採否を決める。

## 足切り
- 変更対象は `task009` のみ。
- 全数 `audit_dir` は使わず、`audit_one` のみ行う。
