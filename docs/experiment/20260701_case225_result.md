## 検証内容

temp ONNX で以下を個別に差し替え、`src.evaluate.scorer.audit_one` で検証した。

## 結論

`k` は入力依存で固定不可だった。`selected_column_u64` 固定は `636 -> 570` まで下がるが足切りを満たさない。`channel_row_u64` の ch0/ch9 は実質ゼロ化可能だが、tensor shape と要素数は変わらないため score cost は下がらない。実 ONNX は変更せず、Kaggle submit もしない。

次は、shape を保つゼロ化ではなく、実際に params 要素数を減らせる table 構造の task を探す。
