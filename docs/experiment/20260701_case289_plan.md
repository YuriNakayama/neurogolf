## Hypothesis

The four trailing zero channels do not need to be materialized in `out20`.
They can be represented by channel-end padding in the final `Pad`:

```text
out20 = Concat(ch0m, zero20, zero20, zero20, mask20, c5u, axis=1)
output = Pad(out20, sp=[0,0,0,0,0,4,10,10], value=0)
```

This preserves the channel layout and the multi-hot semantics rejected in
case259:

- channel 0: `ch0m`
- channels 1..3: zero
- channel 4: `mask20`
- channel 5: `c5u`
- channels 6..9: zero produced by `Pad`

Expected saving: remove `1x4x20x20` uint8 from `out20`, i.e. `1600` memory
bytes.

Expected local cost: `24727 -> 23127`, ratio `1.0692`.
