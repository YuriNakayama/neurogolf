# 20260630 case106 plan - t080 終端 Where の削減検証

## 現状
- floor/Public: 7180.95
- 対象: `task080`
- baseline faithful: cost 17136, params 112, memory 17024, n_fail 0

## 背景
case80 では `lineOrOut` の bool→uint8 Cast を `ReduceMin` 後へ遅延する案を検証したが、`ReduceMin(bool)` が ONNXRuntime で invalid だった。

今回の対象は終端:

```text
Or(seprow, sepcolr) -> sepmask
Where(sepmask, linecu, exp) -> withsep
Where(ingrid, withsep, sentu) -> fu
Equal(fu, lvu) -> output
```

`withsep` と `fu` はどちらも uint8 `[1,1,30,30]` で、各 900B の memory を持つ。

## 仮説
1. `sepmask` による separator overlay が冗長なら、`withsep = exp` にできる。
2. `ingrid` による sentinel mask が冗長なら、`fu = withsep` にできる。

どちらかが faithful に通れば 900B 級の intermediate を消せる。

## 手法
1. `withsep` bypass: final `fu` の入力を `exp` に変更し、`sepmask/withsep` 周辺を実質無効化した候補を監査する。
2. `fu` bypass: final `Equal` の入力を `withsep` に変更し、`ingrid` sentinel mask を無効化した候補を監査する。
3. `n_fail=0` かつ cost 減なら実ファイルに適用して submit する。

## 足切り
- 変更対象は `task080` のみ。
- 全数 `audit_dir` は使わず、`audit_one` のみ行う。
