If a smaller dtype can be used at the output or in the OneHot/CumSum side path
without breaking `Einsum`, task025 could produce a medium relative gain.

2. Test one dtype change at a time:

   - `OneHot` values as float16 if downstream `Einsum` and `CumSum` remain valid;
   - direct uint8/bool feature inputs only if ORT accepts the final `Einsum`.
3. Audit only task025 with `evaluate.audit_one`.
