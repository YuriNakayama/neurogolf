# 20260702 case380 plan - Clip bypass scan

## Context

Accepted Public Score is `7185.88`. Case379 found no current low-risk
standalone candidate after re-running deterministic surgery; only `task285`
remained exact at `gain 0.000938`.

The next target should still follow the logarithmic metric:

```text
gain = ln(old_cost / new_cost)
```

So the useful gate is relative reduction, not absolute cost.

## Hypothesis

Some graphs use `Clip(x, 0, 1)` to convert counts into active masks. If the
clipped tensor is consumed only by operations where any positive value behaves
the same as one, bypassing the `Clip` can remove an intermediate without
changing outputs.

This is not generally true. It must be tested per node because later arithmetic
can depend on the saturated value.

## Method

Run a one-node-at-a-time scan over current ONNX graphs:

1. For every `Clip` node, rewire consumers of the `Clip` output to its data
   input.
2. Remove the bypassed node and stale value_info.
3. Audit only that task with `evaluate.audit_one`.
4. Keep only `n_fail=0` candidates with cost decrease.

Exclude known hidden/runtime risk families:

- `task002` bit-topology local positives;
- `task118` table pruning;
- `task233` uint8 `TopK` value-path compression.

## Gate

Submit only if the best exact low-risk candidate clears:

```text
gain >= 0.020
```

Otherwise write the exact micro candidates to the result and skip submit.
