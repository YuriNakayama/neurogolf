## Hypothesis

`task361` builds center candidates by pooling a `10x10` empty mask and then
slicing the interior `5x5` windows:

```text
MaxPool(empty10_u8, kernel=3) -> empty3_u8 [1,1,8,8]
Slice(empty3_u8, rows 2:7, cols 2:7) -> int_empty_u8 [1,1,5,5]

MaxPool(empty10_u8, kernel=2) -> empty2_u8 [1,1,9,9]
Slice(empty2_u8, rows 3:8, cols 2:7) -> half_empty_u8 [1,1,5,5]
```

Move the crop before the MaxPool:

- integer branch: `Slice(empty10_u8, rows 2:9, cols 2:9) -> 7x7`, then
  `MaxPool(kernel=3) -> 5x5`;
- half branch: `Slice(empty10_u8, rows 3:9, cols 2:8) -> 6x6`, then
  `MaxPool(kernel=2) -> 5x5`.

This preserves each pooled window exactly and removes the wider `8x8` and `9x9`
MaxPool outputs.  A temporary candidate audited exact:
