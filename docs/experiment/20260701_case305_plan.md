## Target

`task328.onnx`

Current local cost scan:

- params: 1940

## Priority rationale

Unlike `task270` and `task280`, the current `task328` graph has a large encoded
template bank:

- `template_bank`: 1717 int64 elements
- `decode_pair_table`: 128 int32 elements
- `base_offsets` / `row_offsets`: lookup indexing

This is a better high-ratio target because a formula rewrite could reduce both
params and memory. The examples show square grids of size 6..18 with nonzero
colors only at corners. The output appears to extend each present corner inward
with a two-cell-period triangular/checker pattern.

## Hypothesis

The template bank encodes a deterministic corner-pattern mask by grid size and
corner occupancy. If the mask can be expressed with coordinate arithmetic and
corner colors, it may remove most of the template bank and clear at least the
2% ratio gate.

## Method

1. Build and verify a NumPy rule against all train/test/arc-gen examples.
2. If exact, estimate whether an ONNX formula can be below 6769.
3. Only edit `task328.onnx` if the estimated graph clearly beats the current
   template-bank implementation.
4. Audit only `task328.onnx` before submit.
