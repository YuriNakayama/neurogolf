## 採否
- **不採用**。
- 5x5 化は正答だが scorer cost が変わらないため採用価値なし。
- node/memory を消せる bypass は全例不一致。

## 結論
`safe_name_71` の pooling 半径は 5x5 までなら正答性に余裕があるが、現行 scorer では kernel size 変更だけでは cost が下がらない。memory を削るには node 自体を消す必要があるが、それは不可。
