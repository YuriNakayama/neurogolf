## Hypothesis

The visible target is the 900-byte `answer_patch`. If any prior cleanup,
index-surgery, or branch-alias candidate became exact against the current
accepted baseline, it could clear the standalone gate because even a 35-byte
reduction is `gain >= 0.020` at this cost.

However, direct dtype conversion is unlikely to help: `answer_patch` is sliced
from float input, so adding `Cast` after the slice keeps the float patch alive
and increases memory unless the producer itself can be changed.

## Method

1. Re-gate existing `task351` cleanup, surgery, and alias scratch candidates.
2. Submit only if a candidate is exact and reaches `submit-candidate`.
3. Otherwise record no-submit and avoid spending a Kaggle slot.
