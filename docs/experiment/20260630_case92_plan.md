# 20260630 case92 plan — t367 の QLinearConv channel/kernel 冗長性を一点検証する

## 背景

- 現 Public best は **7180.87**。
- `task367.onnx` は faithful baseline で cost **25127**（params 5127, memory 20000）。
- Graph は 11 node の極小構成で、入力の color 0/5 channel を crop し、`rect_w` と `fill_w` の 2 段 `QLinearConv` で 20x20 mask を作って出力する。
- 過去ログでは flood-fill/closed-form の手構築は棄却済み。今回の狙いはアルゴリズム置換ではなく、現 QLinearConv の channel/kernel に未使用・同等な面がないかの確認に限定する。

## 仮説

`rect_w` は param 4050 程度を占める。もし `fill_w` 後段で一部 channel が使われない、または特定 output channel を削っても faithful 正答性が保たれるなら、params を下げられる。

## 対象タスク

- `task367`

## 手法

1. initializer shape と `fill_w` の接続を確認し、`rect_w` output channel の寄与を調べる。
2. `/tmp` variant で `rect_w` output channel を一つずつ 0 固定し、faithful `audit_one` を実行する。
3. `fill_w` の kernel/channel についても 0 固定候補を一点ずつ試す。
4. 正答性を保つ channel があれば、重みの縮約または zero 化で cost が下がるか確認する。

## 採否

- faithful `audit_one` が `n_fail=0` かつ cost が 25127 未満なら `task367.onnx` を差し替えて submit する。
- Public Score が 7180.87 を上回れば採用し、DVC push と git commit/push を行う。
- 壊れる候補、または params/memory が下がらない候補は破棄する。
