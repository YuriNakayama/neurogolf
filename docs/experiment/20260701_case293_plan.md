## Background

The last three accepted cycles used crop-at-source rewrites to remove full-size
intermediates. The parallel crop scan found another candidate in `task222`.

- Task: `task222`

## Hypothesis

The graph currently computes a full code image and then crops it:

`1x1 Conv -> Slice([1:15, 1:15]) -> Cast`

The `codef` full `30x30` Conv output has only one consumer, the Slice. Replace the
`1x1` Conv with a no-padding sparse `17x17` Conv that directly produces `14x14`.
Placing the old `1x1` weights at kernel offset `(1,1)` makes output coordinate
`(r,c)` read input coordinate `(r+1,c+1)`, exactly matching the old Slice.

## Method

1. Replace `cw` from `(1,10,1,1)` with sparse `(1,10,17,17)`.
2. Route the Conv output directly to `code32`.
3. Remove the dead Slice and unused crop constants if possible.
4. Audit only `task222.onnx` with `evaluate.audit_one`.
5. Submit and adopt only if Public Score improves.

## Risk

Low. This is a fixed-offset crop equivalence, not a local-value alias. The main tradeoff
is that the sparse kernel adds parameters, so the net gain is smaller than the raw
`30x30 -> 14x14` tensor reduction but still above the normal tiny-cleanup threshold.
