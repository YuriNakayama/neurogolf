# 20260630 case71 plan — t349 の5本QLinearConv detector統合で memory 削減を狙う

## 仮説
t349 は現行 floor cost 26189（params 89 / memory 26100）で、24 node のうち
5本の `QLinearConv(kernel 1x12)` → 5本の `MaxPool` → `Max` が主要構造。
各 detector が同一入力 `ch9_u8` から horizontal position を検出しているため、
出力 channel をまとめた 1本の `QLinearConv`、または detector 数削減で中間 memory を下げられる可能性がある。

## 対象
- task349
- 現行: `data/output/onnx/task349.onnx`
- 現行 cost: 26189（params=89, memory=26100, status=ok）

## 手法
1. floor graph と trace を確認し、memory 支配箇所を特定する。
2. t349 の規則を train/test/arc-gen で確認し、5 detector が本当に独立に必要かを調べる。
3. 独自実装として detector 統合または detector 数削減版を構築し、`audit_one` で faithful 検証する。
4. `n_fail=0` かつ cost<26189 の場合のみ `data/output/onnx/task349.onnx` に反映して submit する。
