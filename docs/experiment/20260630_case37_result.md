## 構造的結論
memory = Σ(全中間テンソルサイズ) のため **single-op net(中間ゼロ)が下限**。floor は各タスクで
single-Conv または最小多ノードに到達済み。独自実装が勝つには「既知の single-Conv より少 param で
同一線形変換を表す新 op 構成」が要るが、Conv params=out_ch×in_ch×k² は I/O 仕様(10ch)で固定され
削減不能。**独自実装の前進には op レベルの新発見が必須で、現プリミティブ範囲では枯渇**。
