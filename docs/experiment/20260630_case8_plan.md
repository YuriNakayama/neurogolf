# 20260630 case8 plan — 全構造クラス網羅監査（floor の最適化フロンティア確定）

## 仮説
hand-golf で floor を破るには「floor が過剰構築している単純構造タスク」を見つける必要がある。
全 400 を既知の単純変換クラスで網羅分類し、各クラスの最高コストを調べれば、floor に残された
hand-golf 余地（単純規則 × 高コスト）を網羅的に特定できる。

## 対象 / 手法
分類クラス: colormap / identity / 幾何変換(flip/rot/transpose) / tiling / mirror-expansion /
flood-fill(enclosed) / k-local(3×3) / gravity / projection。各クラスでコスト順に列挙し、
「単純規則なのに高コスト」のタスクがあれば bit-pack で再設計、無ければ floor がフロンティアと確定。

## 採否
- 過剰構築タスクが見つかれば設計 → faithful → submit。無ければ floor 維持で監査結果を記録。
