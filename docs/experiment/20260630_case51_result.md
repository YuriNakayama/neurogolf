# 20260630 case51 result — t004 CNN を crop+量子化で再挑戦も float hidden で floor 不可（提出なし）

## independent_impl（必須欄）
case49 の uint8 レバーを t004 学習fit CNN（case39 で C=8 厳密 fit 済）に適用:
- **crop 16×16**（t004 max grid=16）で CNN 中間を縮小: case39 64890 → **38343**（cropで1.7×減）。
- だが **float hidden [1,8,16,16] が支配的**（mem 36864）。さらに crop=16 で **nfail=16**
  （16幅 grid が padding context を失い不一致）。
- **CNN は uint8 化が困難**: Conv 重みは float 必須、hidden を QuantizeLinear すると Quantize/
  Dequantize ノードが中間を増やし相殺。flood-fill(MaxPool)と違い uint8 レバーが効かない。

## 確定
- **uint8 レバーは MaxPool ベース（flood-fill）には効くが、Conv ベース（CNN 学習fit）には効かない**。
  Conv は float、量子化は QLinear オーバーヘッド。
- t004 floor 5378 は **単一 Conv + bool ops** で既にタイト。2-Conv CNN（float hidden）は原理的に
  超えられない。t004 は手構築(case37 単Conv不可)・学習fit(case39/51)とも floor 不可で確定。

## 本セッションの技法マップ（どの技法がどこで効くか）
| 技法 | flood-fill | CNN学習fit | 単純局所(単Conv) |
|---|---|---|---|
| crop | ◎ | ○ | △(floor既適用) |
| 単channel | ◎ | × | - |
| **uint8中間** | ◎(4×) | ×(Conv float) | -(floor既bool) |
| uint8出力 | ◎ | ○ | ○ |
→ uint8 が効くのは flood-fill 等 MaxPool/論理演算ベースのタスク。それらは floor も適用済みで parity。

## 恒久方針の遵守
