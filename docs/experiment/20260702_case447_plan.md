# 20260702 case447 plan - bundle-level submit gate

## Context

case446 added known-risk demotion to the one-task `submit gate`, but the actual
`python -m submit submit` path still had a bypass:

1. collect ONNX files;
2. run structural validation with correctness disabled;
3. build `submission.zip`;
4. submit to Kaggle.

That path could still package a changed bundle without checking whether changed
tasks reduce cost, pass task correctness, or hit a known-risk decision.

## Hypothesis

Recent regressions and no-move submits can be reduced by making bundle-level
gate checks part of the submit path itself. The submit command should compare
candidate ONNX files against an accepted baseline directory, run the existing
one-task gate only for changed tasks, and refuse submission if any changed task
is blocked or bank-only.

This saves submit quota while preserving the ability to deliberately override
review-only cases with an explicit flag.

## Method

1. Add bundle diffing by task file digest.
2. Reuse `evaluate_candidate_gate` for each changed task.
3. Accept only `submit-candidate` by default.
4. Allow `review-mid-gain` and `review-known-risk` only with an explicit
   `--allow-review-gate`.
5. Make `python -m submit submit` require `--gate-baseline-dir` or
   `SUBMIT_GATE_BASELINE_DIR` unless `--skip-pre-submit-gate` is explicitly
   passed.
6. Update the scheduled submit script so automated submits abort if no gate
   baseline is configured.
7. Add focused tests for changed-task detection and bundle gate decisions.
