## 試行

`Gather` 用 index initializer の dtype を int64 から int32 に変えた。

| 候補 | params | memory | cost | n_pass | n_fail | filesize | 判定 |
|---|---:|---:|---:|---:|---:|---:|---|
| baseline | 278 | 1633 | 1911 | 265 | 0 | 10101 | ok |
| `render_label_idx` int32 | 278 | 1633 | 1911 | 265 | 0 | 9849 | reject |
| `render_label_idx` + `swap2_idx` int32 | 278 | 1633 | 1911 | 265 | 0 | 9841 | reject |
| Slice/Reshape indices も int32 | invalid | invalid | invalid | 0 | 0 | 9793 | reject |

## 結論

`Gather` の int32 indices は実行可能だが、scorer の `params` は dtype byte数ではなく initializer element数で数えるため cost は変わらなかった。ファイルサイズだけの削減は Kaggle score に効かないため採用しない。Slice 系 indices は int64 と型が混在して session error になった。
