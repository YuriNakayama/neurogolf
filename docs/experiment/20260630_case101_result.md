# 20260630 case101 result - t157 bool Pad 化は ORT 非対応で棄却

## 実装候補
`task157` の `Concat ch1_active_pre + zeros_ch1_rows -> ch1_active10` を `Pad(ch1_active_pre, pads=[0,0,4,0], false)` に置換する候補を一時ファイルで検証した。

## 結果
- baseline: cost 8334, params 351, memory 7983, n_fail 0
- candidate: ONNX checker は通過したが、ONNXRuntime session 作成で失敗
- error: `Type 'tensor(bool)' ... of operator (Pad) ... is invalid`

## 追加確認
`task157` は opset 11。case100 の `task066` は opset 14 で bool `Pad` が実行可能だったため、`task157` の opset を 14 に上げる候補も試したが、既存 `Squeeze` の `axes` attribute が opset 14 仕様と衝突して checker で失敗した。

## 採否
- **不採用**。局所的な 1 node 置換としては実行不可。

## 結論
bool tensor の `Pad` 置換は opset 依存。opset 11 graph では、`Concat` suffix-zero を単純に `Pad` へ置き換える方針は使えない。

## 次の一手
- uint8/float 系、または既に opset 14 以上の graph で、suffix/prefix zero `Concat` を `Pad` 化できる候補を探す。
