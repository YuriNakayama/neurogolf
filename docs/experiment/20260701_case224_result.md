# case224 result: task292 scatter update trim

## 結果

- 対象: `task292`
- baseline: `cost=434` (`params=182`, `memory=252`, `n_fail=0`)
- submit: なし
- 採否: 不採用

## 検証内容

temp ONNX で以下を個別に差し替え、`src.evaluate.scorer.audit_one` で検証した。

- `updates -> pink_source` にして `Mul(update_mask)` を削除
- `update_mask -> ones / left / right / scalar`
- `scatter_idx` の channel 0/1 片側化
- `scatter_idx` 片側化と 1 channel mask の組み合わせ

## 監査結果

| 変更 | cost | n_pass | n_fail | 判定 |
| --- | ---: | ---: | ---: | --- |
| baseline | 434 | 28 | 0 | ok |
| `updates -> pink_source` | 264 | 0 | 28 | reject |
| `mask -> ones` | 442 | 0 | 28 | reject |
| `mask -> left` | 442 | 0 | 28 | reject |
| `mask -> right` | 442 | 28 | 0 | reject: cost worsens |
| `mask -> scalar1` | invalid | 0 | 28 | reject |
| `scatter_idx` channel 0/1 | 1022 | 0 | 28 | reject |
| one-channel idx + one-channel mask | invalid | 0 | 28 | reject |

## 結論

`update_mask=[0,1]` の zero channel は ScatterND の update shape を保つために必要だった。`mask -> right` は正答だが追加 initializer で `434 -> 442` に悪化する。`updates -> pink_source` は `434 -> 264` と大きく下がるが、zero channel が消えて全ペア不正解。実 ONNX は変更せず、Kaggle submit もしない。

次は ScatterND の shape を崩す削減ではなく、同じ shape を保ったまま initializer dtype/encoding を減らせる候補を探す。
