# 2026-06-30 case119 plan

## 方針

ユーザー再確認に従い、優先順位は絶対 cost ではなく相対削減率 `old_cost / new_cost` を重視する。

- floor: Public Score 7181.07
- 優先基準: `ln(old_cost / new_cost)` が直接スコア改善量になる。
- 足切り: `src.evaluate.audit_one` で `n_fail=0` を絶対条件にする。

## 対象

- `task396.onnx`
- baseline: params 119, memory 9489, cost 9608, n_pass 266, n_fail 0

## 仮説

`task396` は `GridSample` 用の横/縦 grid と終盤 recolor 補正を作るために 30x30 級の中間を多く持つ。以下のいずれかが train/test/arc-gen で冗長なら、数 % 以上の相対削減が見込める。

1. 終盤の `Equal(smp,A1)` / `Where(eqs,B1f,smp)` による recolor 補正を消す。
2. 第2色 `B1f` の選択で、`A1` 色除外 `m2` が不要なら `maskedA` を直接使う。
3. `w0f/t0f` や `xnv/ynv` の mask branch を片側化し、横/縦 grid の片枝を削る。

## 手順

1. `task396` の candidate graph を `/tmp` で作り、各候補を `audit_one` で個別検証する。
2. `n_fail=0` かつ cost 減の候補だけ `data/output/onnx/task396.onnx` へ反映する。
3. DVC add 後に submit し、Public Score が 7181.07 を上回れば採用、上がらなければ破棄する。
