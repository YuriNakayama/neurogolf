## 対象タスク

- `task255`

## 手法

1. ORT profile で memory 上位 tensor を確認する。
2. `/tmp` variant で以下を一点ずつ試す。
   - `right_or2 -> right_normal`
   - `top_or2 -> top_normal`
   - `bottom_or2 -> bottom_normal`
   - 各 branch contribution を個別に 0/False 固定
   - `value_info` 全削除
3. faithful `audit_one` が `n_fail=0` の候補だけ cost を比較する。

## 採否

- faithful `audit_one` が `n_fail=0` かつ cost が 23017 未満なら `task255.onnx` を差し替えて submit する。
- Public Score が 7180.87 を上回れば採用し、DVC push と git commit/push を行う。
- 壊れる候補、または cost が変わらない候補は破棄する。
