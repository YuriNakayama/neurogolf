same-colored markers. The current implementation builds an 11x11-ish ray canvas
from many small slices and maxes before applying a dynamic 3x3 stamp kernel.

representation, removing a few ray-row intermediates or `ray_total` could exceed
the 2% gate.

change has a clear expected saving above 138 cost and preserves exact local
