
low-risk edit.

## Label Path Assessment

Replacing `Conv(input, W)` with `ArgMax(input)` would introduce an int64
`[1,1,30,30]` tensor and extra arithmetic before returning to uint8, which is
larger than the current `G + G_u8` path. A `QLinearConv` label path would need
an input-wide uint8 cast from the float one-hot input, also larger than the
current dominant tensor.
