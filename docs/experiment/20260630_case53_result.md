## independent_impl（必須欄）

- **規則を厳密 decode**: 整列した 1-cell ペア間（min〜max 行/列）を 8 で結ぶ。**267/267 一致**。
- **prefix/suffix-OR で非反復実装**: 列=`(CumSum下>0)AND(CumSum上>0)`, 行=同様, OR して間を塗る。

## 学び（dtype レバーの限界）
- **CumSum/Conv 等の数値 op は int32/float 必須で uint8 化不可** → 大 grid で中間が嵩む。
  uint8 レバーは MaxPool/論理(And/Or/Greater)系のみに効く。CumSum ベースは不利。
- 大 grid(26×26)タスクは中間1個でも 2704 要素 = 高コスト。floor の 4-MaxPool(uint8可)が
  CumSum-int32×4 より圧倒的に安い。floor の op 選択が最適。

## 確定
- t350 は非反復で綺麗に解けたが、op 選択(CumSum int32)と大 grid で floor 9036 に届かず。
  floor は MaxPool(uint8化可)を使い同じ「間を塗る」を安く実現。**floor の op 選択が優れる**。
