# 20260702 case473 plan - micro bundle submit cadence gate

## Context

Cases 463-472 produced no Kaggle submit because the post-case414 gate requires
each changed task to be an individual `submit-candidate`. That correctly blocks
local failures and known-risk aliases, but it also prevents exact low-risk
micro candidates from being accumulated into a submit-saving bundle.

The cadence investigation found that `BundleGate.total_gain` already exists but
is not used in bundle acceptance.

## Target

Submit gate only:

```text
backend/src/submit/gate.py
backend/src/submit/__main__.py
backend/tests/unit/src/submit/test_gate.py
```

No ONNX task replacement in this case.

## Hypothesis

Add an explicit opt-in micro bundle lane:

```text
allow_micro_bundle=False by default
allowed decisions: bank-low-gain or submit-candidate only
minimum aggregate gain: 0.015 by default
maximum changed tasks: 5 by default
```

This preserves the hard correctness and risk gates while allowing the main loop
to spend fewer Kaggle submissions when several exact, low-risk micro changes are
available.

## Method

1. Keep default behavior unchanged.
2. Add `BundleGate.is_micro_bundle` and decision `submit-micro-bundle`.
3. Add CLI flags:

```text
--allow-micro-bundle-gate
--micro-bundle-gain
--micro-bundle-max-tasks
```

4. Add focused unit tests that prove:

```text
default low-gain bundle remains blocked
explicit aggregate micro bundle is accepted
review-risk tasks are still blocked
task-count cap is enforced
```

5. Run submit gate unit tests and ruff.
