# 20260630 case90 plan — t209 の uniform scale 判定と crop 周辺を一点検証する

## 背景

- 現 Public best は **7180.87**。
- `task209.onnx` は faithful baseline で cost **32052**（params 185, memory 31867）。
- 過去ログでは t209 は 51149→35838→35498 と手で golf 済みだが、現 floor はさらに 32052 まで下がっている。
- Graph は bbox で色4領域を切り出し、`uniform_s3` と `uniform_s4` を両方計算して `S = s4 ? 4 : s3 ? 3 : 2` を選び、template/kernel と `QLinearConv` で出力を構成している。

## 仮説

`uniform_s3` / `uniform_s4` の両 branch は中間 tensor が大きく memory 支配になりやすい。arc-gen を含む全ペアで実際には片方が不要、または既存 floor に安全な固定 `S` が存在すれば branch 一式を削除して cost を下げられる。

## 対象タスク

- `task209`

## 手法

1. ORT profile で memory 上位 tensor を確認する。
2. `S` を以下の候補に固定または一部 bypass して、prune 後に `audit_one` を実行する。
   - `S = s_pre`（s4 判定削除）
   - `S = THREE_I`
   - `S = TWO_I`
   - `S = FOUR_I`
   - `s_ok_s4 = false` / `s_ok_s3 = false` 相当
3. bbox/crop 周辺に未使用 initializer/value_info/duplicate node があれば、正答性を保ったまま削除する。

## 採否

- faithful `audit_one` が `n_fail=0` かつ cost が 32052 未満なら `task209.onnx` を差し替えて submit する。
- Public Score が 7180.87 を上回れば採用し、DVC push と git commit/push を行う。
- 上がらない、または faithful で壊れる候補は破棄し、結果のみ記録する。
