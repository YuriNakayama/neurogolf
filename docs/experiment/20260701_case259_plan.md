## 仮説

`task367` の出力は実質的に 20x20 label で、

- background: color 0
- fill: color 4 (`mask20`)
- existing gray: color 5 (`c5u`)
- outside 20x20 crop: 全 channel false

で表せる。`label20 = mask20*4 + c5u*5` を作り、pad value 10 で 30x30 に広げてから `Equal(label30, channel_values)` で one-hot 化すれば、`out20` の 4000 byte 中間と `ch0m` を削れる可能性がある。

出力 dtype は bool になるが、既存 solver でも bool one-hot は使われているため、`audit_one` で scorer 互換性を確認する。
