# 20260630 case103 result - t133 correction group は個別にも削減不可

## 実装候補
`task133` の correction overlay 内で、`cc_0..cc_4` の各 group を個別に削れるか検証した。

対象構造:

- `Mul(cc_i, sg_gate_i) -> cu_i`
- `Concat(cu_0..cu_4) -> corr_upd`
- `ScatterND(border_grid, corr_idx_c, corr_upd) -> corr_overlay`

case89 では correction overlay 全体削除が `n_fail=267` だったため、今回は group 単位の冗長性を調べた。

## faithful 検証
baseline:

- cost 33573, params 1278, memory 32295, n_pass 267, n_fail 0

各 group を全ゼロ化:

| 変更 | cost | 結果 |
|---|---:|---|
| `cc_0` zero | 33573 | `n_fail=1` |
| `cc_1` zero | 33573 | `n_fail=1` |
| `cc_2` zero | 33573 | `n_fail=1` |
| `cc_3` zero | 33573 | `n_fail=1` |
| `cc_4` zero | 33573 | `n_fail=1` |

追加で各 group の先頭 1 要素、末尾 1 要素だけをゼロ化したが、全 group で `n_fail=1`。

## submit
- なし

## 採否
- **不採用**。
- correction group は全体だけでなく、個別 group の端要素まで faithful 上必要。

## 結論
t133 の correction table は小さい例外補正の集合だが、各 group が少なくとも 1 例に効いている。単純な group 削除・suffix/prefix 切り詰めでは cost 削減できない。
