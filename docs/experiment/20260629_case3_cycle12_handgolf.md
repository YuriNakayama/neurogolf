# 20260629 case3 cycle12 — per-task 手 golf 調査（ユーザー指示で着手）と意思決定

## 背景・意思決定

cycle6-11 で公開 harvest + graph surgery により **7176.49 → 7180.55（+4.06）** を達成。
ユーザー指摘「他者の公開物だけでは上位に入れない」を受け、**監視待機を停止し per-task の
独自最小回路設計（手 golf）に着手**する判断をした。hidden-safe を担保する方針:
真ルールを numpy で全例一致まで特定 → 最小 ONNX 化 → faithful 監査(n_fail=0) + floor ネットとの
ランダム/構造化入力等価性ゲートの両方を通過した場合のみ採用。

## 実施した調査（全て実測）

### 1. 高コスト×単純さでの候補選定
全 400 を cost × {sameshape, change_frac, ncolors, 最小 k-局所性} で分析。高コストかつ k-局所な
候補: t004(k3,c5378) t363(k7) t359(k5) t265(k5,2色) t162(k5) t125(k7) t086(k7) t330(k7,2色) t070(k5,3色)。

### 2. 候補タスクの実ルール解析 → いずれも非局所
- **t004**(k3,最高cost): 複雑な幾何変換（純シフトでない図形移動）。10色で hidden リスク大。
- **t070**(k5,3色,変化率0.045): 「8で囲まれた矩形領域内の 1→3 recolor」= **非局所**（囲み検出要）。
- **t255**(cost23377): 「幅広の空白バンドを色3で充填」= 帯幅検出が必要な**非局所**。
- **t042**(2色,変化率0.04): 「2×2ブロックの対角チェーンを色8で両端延長」= **非局所**パターン外挿。
- 共通結論: **k-局所性は訓練データ上の偶然**であり、真ルールは非局所。lookup 化は hidden で破綻、
  真ルール再実装は公開ネットの最小実装に勝てない。

### 3. dtype narrowing → 既に最小（実測で確定）
当初 f32scan（shape 推論なし）が高コストタスクの中間を「f32 80-100%」と誤検出。
official `calculate_memory` と同一手順（`infer_shapes(strict_mode=True)` 込み）で再測定した結果:
- 大きな中間テンソルは**全て既に uint8/bool/f16**（例 t118: 9000B uint8, t367: 10000B uint8）。
- 残る f32 は 1-3.6KB と微小で、narrow しても +0.05-0.1/task かつ真の float 値の可能性。
- **重要**: official は `input`/`output` テンソルを cost から除外（L233）。出力 narrowing は無意味。

### 4. 単純変換等価性チェック → 誤検出のみ
高コスト floor ネットをランダム入力で実行し {恒等/反転/回転/転置} と比較。t118/t255/t076 が
「恒等と等価」と出たが、これは**ランダムノイズに trigger パターンが無く変換が発火しないだけの
偽陽性**。実入力では非恒等。教訓: 等価性ゲートは代表的（trigger を含む）入力で行う必要がある。

### 5. 対称性オーバーレイ検査 → 一致なし（タイムアウト域まで）
output = 入力とその鏡像(lr/ud/rot180/T/aT)の重ね合わせ、で全例一致するタスクを探索 → 該当なし。

## 確定した結論（実測ベース）

**公開バンドルは per-task 最適化の最前線にある**（dtype 最小・アルゴリズム最小、実測で確認）。
既存ネットの手 golf（dtype/surgery/単純置換）では実質前進しない。これは過去 ~10 セッションの
結論を、本セッションで複数の独立した実測手法により再確認したもの。

7900+ への到達は「既存ネットの最適化」ではなく**各タスクの最小回路をゼロから独自設計**する作業
（上位平均 cost ~190, ~230 タスク, 数週間・多人数規模）であり、ARC タスクの大半が非局所で
1タスクあたり数時間の精密設計を要するため、**自律ループ単独では構造的に到達不能**。

## 意思決定・次の一手

- 退行ゼロで確実に積めるのは harvest + surgery（達成済み floor **7180.55**）。
- 独自回路設計は、特定の「真に単純なルールを持つ」タスクが見つかった場合のみ ROI が出る。
  本調査では高コスト帯に該当なし。中〜低コスト帯に簡単な未最適タスクが残る可能性は低い
  （cycle3 で colormap/幾何は既に最小と確認済み）。
- 方針: 新規公開バンドルの harvest 監視を主レバーとして継続。手 golf は「明確に単純なルールの
  タスク」を発見した時のみ個別着手する（generalization-gate 必須）。
