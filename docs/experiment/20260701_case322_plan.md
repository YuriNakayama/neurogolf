## Hypothesis

The tail currently materializes a five-channel crop with an explicit zero
channel, then pads it to the final 10-channel output:

```text
Concat(bg, x1, x2_shift, zero, x4_shift) -> crop_output
Pad(crop_output, out_pad) -> output
```

Only four live channels are needed. Replace the tail with:

```text
Concat(bg, x1, x2_shift, x4_shift) -> packed_live
QLinearConv(packed_live, one-hot 4ch->10ch weights, pads=final output pad) -> output
```

This preserves the output channel layout while removing the explicit zero
channel tensor and moving final padding into the projection.
