Some accepted ONNX graphs still carry stale `value_info` entries for tensors
that are no longer produced. Pruning those entries is semantics-preserving and

candidate; smaller exact prunes should be accumulated in the micro backlog.

4. Submit only if a candidate reaches `submit-candidate`.
5. Bank exact low-gain candidates for a future reviewed micro bundle.
