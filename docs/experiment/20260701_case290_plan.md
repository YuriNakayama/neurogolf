# case290 plan: task209 crop-at-source label Conv

## Background

Recent accepted cycles improved Public Score, but the per-cycle gain is too small when
we only remove small aliases or scalar intermediates. Because the metric is logarithmic,
the planning target should be relative reduction:

`delta_points = ln(old_cost / new_cost)`

So this cycle raises the candidate bar from "safe small cleanup" to a structural rewrite
with an expected cost ratio above 1.05, while preserving `n_fail=0`.

## Target

- Task: `task209`
- Current local cost: `31187`
- Expected cost after rewrite: about `29487`
- Expected ratio: `31187 / 29487 = 1.0577`
- Expected point delta: `ln(1.0577) = 0.056`

## Hypothesis

The current graph first computes a full `30x30` label image with a `1x1` Conv, casts it
to uint8, and then slices the top-left `20x20` crop. This materializes a larger
intermediate than the solver actually consumes.

Replace the `1x1` label Conv with an `11x11` sparse Conv whose only non-zero taps are at
the top-left kernel location. With no padding, a `30x30` input produces `20x20` output.
This is semantically equivalent to the old full label Conv followed by a top-left
`20x20` slice:

- old: `label_full[:, :, 0:20, 0:20]`
- new: sparse kernel reads the same input cell at each output coordinate

The rewrite depends on static geometry rather than local-value coincidence, so it avoids
the hidden-unsafe alias pattern that caused case287 to regress.

## Method

1. Change `label_conv_w_209` from shape `(1,10,1,1)` to sparse `(1,10,11,11)`.
2. Route the following Cast directly to `task209_qlabel_u8`.
3. Remove the now-dead Slice node and its constants if unused.
4. Audit only `task209.onnx` with `evaluate.audit_one`.
5. Submit and accept only if Kaggle Public Score improves.

## Larger-Gain Rule Going Forward

Use these filters before spending a cycle:

- Prefer candidates with `old_cost / new_cost >= 1.05`.
- Prioritize deleting whole tensors, channel blocks, full-size masks, or dtype-expanded
  intermediates.
- Avoid locally true aliases unless there is a shape/operator proof independent of the
  public examples.
- Keep tiny cleanups only as fallback when no structural candidate is ready.
