## Hypothesis

`task364` crops a 20x22 region, computes two lookup masks (`gA`, `gB`), runs two
parallel 7-step `MaxPool`/`Mul` propagation chains, combines the two propagated
fields, and restores the 30x30 one-hot output.

Each 20x22 uint8 intermediate costs 440 bytes. If the 7th propagation step is
redundant for public examples, deleting one or more tail nodes would provide a
measurable relative reduction:

- one deleted tail tensor: about 1.9% memory reduction
- one deleted `MaxPool` + one deleted `Mul`: about 3.8% memory reduction
- symmetric A/B tail trim: about 7.6% memory reduction

## Method

1. Probe only temporary `task364` ONNX copies.
2. Replace final chain consumers with earlier propagation tensors and delete the
   now-unused suffix nodes.
3. Test one depth change at a time with `audit_one` for `task364`.
4. Apply and submit only if `n_fail=0` and cost decreases.
