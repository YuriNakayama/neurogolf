- **敗因（構造的・決定的）**: Conv-ReLU-Conv は **2 つの hidden 中間テンソル [1,C,30,30]** を持ち、
  memory=Σ中間 に加算される（30×30 静的）。最良の int8 量子化でも:
  - C=3: uint8 mem ~5400 ≈ floor だが **C=3 は厳密 fit 不可**（0/265）。
  - C≥4: 厳密 fit 可だが uint8 mem ≥7200 > 5378。さらに Quantize/Dequantize ノードが中間を増やす。
  - ∴ **2層 CNN は t004 で floor 5378 を構造的に下回れない**。

## 確定（t004 の3手法すべて失敗 = strategy.md 諦め基準を満たす）
- 手構築（single-Conv, case37）: 非一様変形で線形分離不可 → fit せず。
- 学習fit（本 case39）: 厳密 fit 成功するが 2-hidden 中間 memory が floor 超過。
- プログラム合成: 同じ局所非線形規則を表すには Conv-ReLU-Conv 相当が必要で学習fitと同構造 → 同じ memory 障壁。
- → t004 は独自実装で floor 不可と3手法で実証。

## 一般知見（学習fit の構造的限界）
- 学習fit は**規則の厳密学習には成功する**（重要: 「解けない」のではない）。
- だが floor が bool/int 中間（Where/Equal/Less が 1byte）で memory を抑えるのに対し、
  CNN は float/quant 中間が嵩む。**memory=Σ中間 モデル下では「中間の少ない・低 dtype の floor net」が有利**。
- 学習fit が勝てるのは「floor が多層 float 中間を持ち、より少中間の CNN で置換できる」タスク。
  t004 floor は既に bool 中間主体で最適化済みだった。
