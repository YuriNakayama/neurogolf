# 20260702 case447 result - bundle-level submit gate

## Result

No Kaggle submit. Implemented a submit-path guardrail so bundle submissions
cannot bypass the one-task correctness/gain gate by default.

## Changes

Added bundle gate support in `backend/src/submit/gate.py`:

- `changed_candidate_files(files, baseline_dir)` compares candidate and
  baseline task files by SHA-256 digest.
- `evaluate_bundle_gate(files, baseline_dir, task_dir)` gates only changed
  tasks.
- `BundleGate` reports changed tasks, blocked tasks, total expected gain, and
  bundle decision.

Updated `python -m submit submit`:

- new `--gate-baseline-dir` option, also configurable with
  `SUBMIT_GATE_BASELINE_DIR`;
- new `--gate-task-dir` option, also configurable with `SUBMIT_GATE_TASK_DIR`;
- default behavior now refuses ungated submits unless
  `--skip-pre-submit-gate` is explicitly passed;
- default accepted changed-task decision is only `submit-candidate`;
- `--allow-review-gate` explicitly allows `review-mid-gain` and
  `review-known-risk`;
- blocked/banked/no-change bundles exit before `submission.zip` generation and
  Kaggle submission.

Updated `infra/scripts/submit_once.sh`:

- automated scheduled submits now require `SUBMIT_GATE_BASELINE_DIR`;
- the script passes `--gate-baseline-dir` and `--gate-task-dir` to
  `python -m submit submit`;
- if no baseline is configured, the script aborts instead of spending a submit
  on an ungated bundle.

## Decision

Adopt. This does not increase Public Score directly, but it addresses the
observed cause of wasted or regressing submit cycles: local structural
validation was able to produce a Kaggle submission without changed-task
correctness/gain/risk evidence.
