## Hypothesis

The largest apparent reductions are likely semantic, not mechanical:

- removing `ScatterElements` changes the special column-gap correction;
- bypassing `Xor` changes the hidden-fill mask;
- removing `Pad(fill)` breaks the full-size `Where` condition;
- replacing the channel-8 float slice is unlikely because `Slice` already crops
  directly to channel 8 and a 17x17 region.

The only bounded low-risk probes are index/pad dtype or optimizer cleanup checks
against the current baseline. They are expected to be no-submit unless they are
both exact and cost-reducing.
