# case243 result: task042 small conv kernel crop

## 結果
- 対象: `data/output/onnx/task042.onnx`
- baseline: `cost=3110` (`params=318`, `memory=2792`, `n_pass=266`, `n_fail=0`)
- 採否: 不採用
- Kaggle submit: なし

## 検証内容
kernel の非ゼロ bbox を確認した。

| kernel | shape | 非ゼロ bbox | 削減余地 |
|---|---:|---:|---:|
| `m1_pair_w_3` | `(2,1,4,4)` | rows `0:3`, cols `0:4` | 4 spatial cells |
| `m1_shift_w_5` | `(2,1,4,4)` | full | 0 |
| `m2_w_7` | `(1,1,9,9)` | full | 0 |
| `m3_w_9` | `(1,1,11,13)` | full | 0 |

`m1_pair_w_3` のみ crop 候補を probe した。

| 候補 | cost | n_pass | n_fail | 判定 |
|---|---:|---:|---:|---|
| baseline | 3110 | 266 | 0 | ok |
| `m1_pair_w_3 4x4 -> 3x4` direct | - | 0 | 0 | session error: `Max` shape mismatch |
| `m1_pair_w_3 4x4 -> 3x4` + output Slice top-left | - | 0 | 0 | session error: `Max` shape mismatch |
| `m1_pair_w_3 4x4 -> 3x4` + output Slice bottom-right | - | 0 | 0 | session error: `Max` shape mismatch |

## 結論
唯一の crop 候補は `m1_pair_w_3` の下端ゼロ行だけで、後段 `ConvTranspose` の output shape と最終 `Max` 合流 shape が崩れる。補正には追加の shape 調整ノードが必要だが、削減できるのは spatial 4 要素 × 2 channel 程度で、追加 memory に対して割に合わない。`n_fail=0` かつ cost 改善候補がないため、ONNX は変更せず submit しない。

## 次の一手
kernel crop は、出力 shape がそのまま保てる `QLinearConv` pads 補正型、または case241 のように多数回再利用される sparse kernel を優先する。単発かつ shape 補正が必要な小 kernel は見送る。
