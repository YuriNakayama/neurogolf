## 仮説
`task066` の終端は `hidden4` を bool zero initializer 2 本で `Concat` し、最終 `Where` 用の 30x30 mask に拡張している。

- `hidden_right`: bool `(1,1,20,10)` all zero
- `hidden_bottom`: bool `(1,1,10,30)` all zero
- `Concat hidden4 + hidden_right -> hidden_wide`
- `Concat hidden_wide + hidden_bottom -> hidden30`

この 2 段 `Concat` を 1 個の `Pad(hidden4, pads=[0,0,0,0,0,0,10,10], false)` に置換できれば、zero mask params と intermediate memory を削れる可能性がある。

## 手法
1. `task066.onnx` の終端 2 `Concat` を `Pad` 1 node に置換する候補を作る。
2. `src.evaluate.scorer.audit_one` で `task066` の全ペア完全一致を確認する。
3. `n_fail=0` かつ cost 減なら Kaggle submit。
4. Public Score が 7180.89 を上回れば採用し、上がらなければ DVC checkout で破棄する。

- 全数 `audit_dir` は使わず、`audit_one` で個別監査する。
