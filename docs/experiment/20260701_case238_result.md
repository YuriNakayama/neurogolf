## 結論
13 長 kernel の端 0 は単なる余白ではなく、Conv stride/Slice 位置合わせに必要だった。length 12 でも完全一致を保てず、length 11 は output shape か正答性を壊す。`n_fail=0` かつ cost 改善の候補がないため、ONNX は変更せず submit しない。

## 次の一手
1D kernel は位置合わせの影響が強く、単純 trim は危険。次は 2D kernel/table で nonzero bbox より外側に安全余白があるものを優先する。
