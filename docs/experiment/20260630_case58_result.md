## independent_impl（必須欄）
未試行角度「single-Conv floor の kernel が実 reach より過大なタスク」を精査:
- 全 single-Conv floor の kernel サイズ vs 規則 reach を照合 → t032(9×1, cost910) を候補抽出。
- **t032 = gravity**（各列の非ゼロセルが下に落ちる、最大3個stack）。266/266。max grid 6×6。
- **gravity は非線形**（列に最大3個 stack、順序依存）なのに floor は **単一 9×1 Conv で nfail=0**。
  → floor は線形でない gravity を 9×1 Conv で正答実装（限られた入力パターンに対し成立）。
- **kernel 縮小可否**: floor の 9×1 weight は **全9行が非ゼロ**（pads[5,0,3,0]で全列を参照）。
  全行が gravity 計算に必要。params=10×10×9=900 は I/O 10色で構造的最小。**縮小不可**。

## 確定
- t032 の 9×1 Conv は kernel 全行・全 channel を使う**最小 single-Conv gravity** 実装。過大でない。
- 他の「過大疑い」候補(t097/171 等 3×3)も border/local 規則で 3×3 が必須。過大なし。
- **single-Conv floor の kernel は全て真に必要なサイズ**。コミュニティは kernel も最小化済み。
- gravity を独自に多 op（反復落下=6 iter）で組むと single-Conv 910 を超える。floor が最適。
