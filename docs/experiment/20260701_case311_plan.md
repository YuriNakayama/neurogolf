## Hypothesis

`task400` ends with a dynamic nearest-neighbor `GridSample`:

```text
GridSample(input, grid) -> patch [1,10,5,5]
Pad(patch) -> output [1,10,30,30]
```

The grid is generated from the location of the object marker:

```text
arow = 23 - row
acol = 23 - col
gx = normalized(acol) - [0..4] * 2/29
gy = normalized(arow) - [0..4] * 2/29
```

With `align_corners=1` and `mode=nearest`, this should equal:

```text
input[:, :, arow-i, acol-j] for i,j in 0..4
```

So the float grid path may be replaceable by a dynamic negative-step `Slice`:

```text
Slice(input, starts=[arow, acol], ends=[arow-5, acol-5],
      axes=[2,3], steps=[-1,-1]) -> patch
Pad(patch) -> output
```

Expected saving if exact:

- remove float grid construction tensors: `gx`, `gy`, `gxu`, `gyu`, `grid`;
- remove `offx/offy/tw/one/a3` params;
- expected cost roughly `1866 -> 1227`, ratio about `1.52`.
