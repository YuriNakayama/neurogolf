found candidates that were either:

as a reason to spend a Kaggle submit on a weak or risk-listed candidate.

## Hypothesis

Increasing independent search workers should raise the chance of finding one
candidate that passes the current gate without relaxing the post-regression
safety rules.

Priority remains relative reduction:

```text
gain = ln(old_cost / new_cost)
```

Target candidates:

```text

not in hard/review/runtime risk sets
standalone gain >= 0.020, or clean micro-bundle gain >= 0.015
mechanical low-risk explanation: stale metadata, broadcast initializer shrink,
or conservative graph cleanup
```


All workers write only under `/tmp/neurogolf_agent_R*`, do not commit, do not
submit, and do not modify tracked repo files.

## Parent Gate

The parent process owns all repo changes, submission bundle assembly, Kaggle
submit, adoption/rejection, result docs, commit, and push.

Submit only if `backend/src/submit/gate.py` returns:

```text

```

Otherwise bank exact low-gain material and record a no-submit result.
