## 優先度変更

ユーザー再確認に基づき、今後の探索優先度を以下へ変更した。

- 絶対 cost ではなく `old_cost / new_cost` を重視する。
- スコア改善は `ln(old_cost / new_cost)` なので、同じ固定 cost 削減なら小〜中 cost タスクの方が効率がよい。
- ただし低 cost 単純タスクは最小化済みが多いため、狙い目は「中〜高 cost で 30x30/20x20 中間を 1 個以上消せる構造」。

この方針で全 task の no-correctness cost を再取得し、低 node / 中 cost / memory 支配の `task074` を選んだ。
