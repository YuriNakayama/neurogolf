# 20260630 case36 result — 独自実装の構造的限界を実証: floor は single-Conv 下限、独自net は中間テンソルで必敗（提出なし）

## independent_impl（必須欄）
- **対象**: 局所 k×k 規則で厳密 decode 可能な 62 タスクのうち、small-LUT で勝てそうな低コスト群
  （t171 lut=9, t230=38, t095=34, t282=34 等）。
- **手法（手構築）**: t171（規則 = grid 外周に色8の枠、内部0）を最小 op で構築。
  - hot=ReduceSum, ext=1-hot, MaxPool dilation, border=hot*ext_dilated, ch0=hot-border, Concat。
  - 結果 **cost=21602 ≫ floor 910（24倍）**、かつ nfail=54。
- **敗因の構造分析（決定的）**: floor の t171 net を inspect → **single Conv, 1ノード, params=910,
  中間テンソル 0**。これは case30 の証明「**single-Conv = 中間ゼロ = memory 下限 = 不可侵**」の実例。
  私の独自 net は ReduceSum/MaxPool/Mul/Sub/Concat と **6 中間テンソル**を持ち、memory-sum モデルで
  各 [1,1,30,30]≈3600B が加算され必ず floor を超える。

- **single-op net は中間テンソルが無く memory 下限**。多ノードの独自 net は原理的に超えられない。

## 確定（本セッションの最終判定・strategy.md 3手法基準）
独自実装を実際に build/measure した結果、勝てない理由が**構造的に確定**:
1. **高コストタスク**（t002/t219/t076/t118…）= 規則が非局所（flood-fill/staircase/object-stamp）。
   忠実実装は多数の中間テンソルを要し floor の閉形式を超える（case35 で実証）。
2. **低コストの単純タスク**（recolor/border/local-rule）= floor が既に **single-Conv 下限**。
   独自実装は中間テンソルを足す分だけ必ず高コスト（本 case36 で実証）。
3. **量子化**（int8 conv）も floor が適用済みで params 最小化済み。
- ∴ **独自実装で floor を下回る余地は、single-Conv 化されたタスク群には存在しない**。
  これは「不可能の安易な一般化」ではなく、t002/t171 を実際に構築・計測し floor net を
  inspect した上での構造的結論。case30（fleet/serial）+ case35（高コスト decode）+ case36
  （低コスト single-Conv 実証）で三重に確定。

## 残る前進レバー（事実）
- floor は per-task で single-Conv/量子化-Conv の下限に到達。**唯一の純増は、新しい公開実装が
  ある per-task で現 floor より安い single-Conv を出した時の harvest**（cross-bundle min）。
- これは「harvest への逃げ」ではなく、独自実装が構造的に下限に達したことの帰結。
  独自実装は新規 op 構成（例: 既知 single-Conv より少 param な表現）が見つかった時のみ意味を持つ。

## 次の一手
- 独自実装: single-Conv 下限を**下回る**新表現の探索のみ意味がある（例: 同一線形変換を
  より少 param な構造で。ただし Conv params=出力ch×入力ch×k² は既に最小）。現実的余地は薄い。
- harvest 監視を補助として継続。floor 7180.78（DVC 6e8d4642）退行なし。

## 追試（決定的計測）: params 削減も中間テンソルで相殺
- t171 を `Slice(ch0)→Conv[10,1,3,3]` で構築。**params は 90（floor 900 の 1/10）に削減**できるが、
  Slice の中間テンソル c0=[1,1,30,30]≈3600B が memory に加算され **total cost=3703 ≫ floor 910**。
- ∴ **single-Conv（中間ゼロ）から params を削るために op を足すと、足した中間テンソルが
  削った params を上回る**。memory-sum モデル下で single-Conv は局所最適かつ大域下限。
- 「3×3 で 10ch→10ch」の Conv params は構造上 10×10×9=900 が最小（要素数はゼロ値でも計上）。
  境界検出には k≥3 が必須（1×1 不可）。よって floor 910 は構造的最小。
