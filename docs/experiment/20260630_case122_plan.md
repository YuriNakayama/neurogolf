# 2026-06-30 case122 plan

## 方針

- floor: Public Score 7181.08
- 優先基準: 相対削減率 `old_cost / new_cost`。
- 対象: `task358.onnx`（cost=4486）

## 仮説

`task358` には Cast チェーン冗長がある:
- `Cast(row_ids_f → bool) → row_nonzero` [22]
- `Cast(col_ids_f → bool) → col_nonzero` [23]
- `Cast(row_nonzero → uint8) → row_nonzero_u8` [24]
- `Cast(col_nonzero → uint8) → col_nonzero_u8` [25]

`row_ids_f` は Einsum `nchw,c,nh->nw` で得られる色インデックス (0〜9 の非負整数)。
`Cast(float→bool)` 経由の 2 段 Cast は、`Cast(float→uint8) → Min(uint8, 1)` に置換可能。
`row_ids_u8` / `col_ids_u8` は既に独立した Cast[30]/[31] で存在する（GatherElements 用）。
これを再利用すれば bool 中間テンソル `row_nonzero`(30 bytes)・`col_nonzero`(30 bytes) を除去できる。

コスト見積もり: -60 bytes memory + 1 byte params (new one_u8 initializer) = -59 cost。

## 手順

1. Cast[22]/[23] を削除（float→bool）。
2. Cast[30]/[31]（float→uint8）を前方へ移動。
3. Cast[24]/[25] を `Min(row_ids_u8, one_u8)` / `Min(col_ids_u8, one_u8)` に置換。
4. `one_u8 = 1` uint8 初期化子を追加。
5. `audit_one` で n_fail=0 かつ cost<4486 を確認後 `data/output/onnx/task358.onnx` に反映。
