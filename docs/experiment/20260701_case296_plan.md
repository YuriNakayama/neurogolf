## Hypothesis

`task158` starts by converting the full one-hot input into a color label grid:

```text
Conv(input, w_color 1x1) -> color_f [1,1,30,30]
Cast(color_f) -> color_u8 [1,1,30,30]
Slice(color_u8, rows 0:26, cols 0:25) -> color_work [1,1,26,25]
```

`color_u8` has a single consumer, and all downstream logic uses only
`color_work`.  The crop is top-left aligned, so the Conv can emit exactly the
cropped region by expanding the kernel from `1x1` to sparse `5x6` with the
original weights at offset `(0, 0)`.

Expected effect:

- remove full `30x30` float `color_f` and uint8 `color_u8`;
- replace them with `26x25` float `color_f` and direct Cast output
  `color_work`;
- remove the Slice node;
- params increase by `+290` (`10 -> 300`) but memory should drop by about
  `1250`, for net cost improvement about `960`.

This is not the desired `1.5x` class, but it is proof-grade and materially
larger than tiny alias-only edits.  Since no larger verified candidate is
available, use it as the next safe cycle while continuing large-delta search.
