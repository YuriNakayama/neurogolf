## Priority rationale

The scoring improvement is `ln(old_cost / new_cost)`, so this cycle keeps the
case303 ratio gate:

- normal implementation threshold: `old_cost / new_cost >= 1.02`
- for `task280`, target cost must be below `6039 / 1.02 = 5918.6`
- preferred target for a semantic rewrite: below `6039 / 1.05 = 5751.4`

`task280` is a medium-cost graph with 112 nodes and many branch/probe operators
(`Cast`, `Add`, `Or`, `GatherND`, `Concat`, `TopK`). This is a better target
than byte-level cleanup because removing only a few medium tensors can cross the
2% gate.

line/rectangle pattern from marker defects. The current graph likely derives the
direction and anchor through several branch tensors. If the rule can be rendered
more directly, we may remove enough temporary tensors or selector parameters to
save at least 121 cost.

4. Audit only `task280.onnx`; submit only if `n_fail=0` and cost improves past
