- **calculate_memory は全 tensor（各 node 出力 + value_info）のサイズを総和**する（peak/max ではなく
  **sum**）。`return sum(tensor_memory.values())`。→ **memory ∝ tensor 総数 × 各サイズ**。
- これが explicit hand-construction が爆発する正確な理由（各ノードが output tensor を総和に加算）。
  また floor の高コスト net（t233 mem=56151）は多数の [5,324] 中間 tensor の総和。
