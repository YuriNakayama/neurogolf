## 試行

`task316` の TopK score を作る Cast を `FLOAT16` から `UINT8` に変更した。

```text
occupied_b -> Cast(float16) -> scores
```

を:

```text
occupied_b -> Cast(uint8) -> scores
```

へ変更した。TopK の `topvals` は未使用で、必要なのは `topidx` だけなので、0/1 ranking が保たれれば
成立するという仮説だった。

## Kaggle 結果

提出は `SubmissionStatus.ERROR` で Public Score が出なかった。ローカル `audit_one` では
ONNX Runtime が uint8 TopK を実行できたが、Kaggle 側の validator/runtime ではこの dtype の
TopK が受理されない可能性が高い。

## 結論

不採用。TopK score dtype を uint8 にする変更は、ローカルで通っても Kaggle 側で ERROR になる。
今後は TopK 入力 dtype を float/float16 から整数へ落とす候補は避ける。TopK の値が未使用でも、
公式 runtime の dtype 制約を優先する。
