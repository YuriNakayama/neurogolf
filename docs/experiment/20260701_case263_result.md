## 試行

`labels` Conv の `w[1,10,3,3]` は中央1x1だけが非ゼロだったため、kernelを1x1へcropし、padsを調整する候補をprobeした。

| pads | 結果 | 判定 |
|---|---|---|
| `[16,16,1,1]` | Resize shape inference error: output 120 vs expected 30 | reject |
| `[17,17,0,0]` | Resize shape inference error: output 120 vs expected 30 | reject |
| `[16,16,0,0]` | Resize shape inference error: output 115 vs expected 30 | reject |
| `[17,17,1,1]` | Resize shape inference error: output 120 vs expected 30 | reject |
| `[16,16,2,2]` | Resize shape inference error: output 120 vs expected 30 | reject |

## 結論

非ゼロ値だけを見ると1x1化できそうだが、元の3x3 kernelは dilation/stride と組み合わせて Conv の6x6中間形状を作る役割も持っている。1x1化すると Conv output が大きくなり、後段 `Resize` の shape が崩れる。Slice等で戻すと中間が増えて削減にならないため、ONNXは変更せずsubmitしない。

kernel crop は非ゼロbboxだけでなく、Conv output shapeを追加Sliceなしで保てるかを先に判定する。padsで吸収できないcrop候補は優先度を下げる。
