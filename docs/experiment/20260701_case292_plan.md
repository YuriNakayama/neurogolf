# case292 plan: task396 crop-at-source label Conv

## Background

case290 and case291 showed that the best way to increase per-cycle delta is to delete
whole full-size intermediates with a shape/operator proof. This cycle applies the same
crop-at-source pattern to another task found by graph scanning.

## Target

- Task: `task396`
- Current local cost: `9608`
- Expected new cost: about `8084`
- Expected ratio: about `1.19`

## Hypothesis

The graph currently computes a full label image and then crops it:

`1x1 Conv -> Cast -> Slice(18x18)`

The Slice selects the top-left `18x18` region of a `30x30` tensor. A no-padding `13x13`
sparse Conv whose only non-zero taps are at the top-left kernel location produces the
same `18x18` label tensor directly:

- old: `label_full[:, :, 0:18, 0:18]`
- new: sparse Conv output at each coordinate reads the same input cell

This should remove the `30x30` float and uint8 intermediates while adding only sparse
kernel parameters.

## Method

1. Replace `wenc` from `(1,10,1,1)` with sparse `(1,10,13,13)`.
2. Route the Cast directly to `gu18`.
3. Remove the dead Slice and unused crop constants.
4. Audit only `task396.onnx` with `evaluate.audit_one`.
5. Submit and adopt only if Public Score improves.

## Risk

Low. This is the same fixed top-left crop equivalence as case290/case291 and does not
depend on local example values.
