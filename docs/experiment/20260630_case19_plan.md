# 20260630 case19 plan — dtype/fp16 削減で params・memory dominated net を golf

## 仮説
cost=params+memory。params-dominated net（単一 Conv, mem=0）を fp16 化すれば params が半減するか。
memory-dominated net（t233）の peak テンソルを小 dtype 化すれば memory が減るか。

## 手法
- calculate_params のソース確認（element count か byte count か）。
- t233 peak テンソル dtype を runtime trace で確認、fp32→fp16 の value-preserving 余地を調査。
