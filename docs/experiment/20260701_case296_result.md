## Change

`task158` previously computed a full `30x30` color label grid and then cropped it:

```text
Conv(input, w_color 1x1) -> color_f [1,1,30,30]
Cast(color_f) -> color_u8 [1,1,30,30]
Slice(color_u8, rows 0:26, cols 0:25) -> color_work [1,1,26,25]
```

The crop is top-left aligned and `color_u8` was consumed only by that Slice.
The adopted graph expands `w_color` from `1x1` to sparse `5x6`, updates
`kernel_shape`, routes the Cast directly to `color_work`, and removes the Slice.

This increased params by `+290` but removed the full `30x30` uint8 label tensor
and shrank the float Conv output, giving a net `-1610` cost reduction.

The large-delta search found no proof-grade `old_cost/new_cost >= 1.5` candidate,
