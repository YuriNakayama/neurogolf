## Hypothesis

The tail currently builds a 9-channel tensor with repeated explicit zero
channels, then pads to the final output:

```text
BitwiseXor(ch3, ch3) -> zero
Concat(bg, zero, zero, ch3, zero, zero, ch6, ch7, ch8) -> small
Pad(small, out_pads) -> output
```

Replace this with a packed tensor of the five live channels and a one-hot
`QLinearConv` projection:

```text
Concat(bg, ch3, ch6, ch7, ch8) -> packed_tail
QLinearConv(packed_tail, one-hot 5ch->10ch weights, pads=final output pad) -> output
```

The projection preserves the original output channel layout while removing the
four explicit zero-channel references and the final separate `Pad`.
