## 仮説

`task390` は 15x15 の red core に対し、`gray_indices (1,1,11,8,4)` と `gray_updates (1,1,11,8)` を `ScatterND` して外周 gray 線を作る。`gray_indices` だけで 2816B の int64 params を占めるため、11 row のうち 1 row でも faithful/test/arc-gen 上で不要なら、indices params 256B と対応 update memory を同時に削れる。

特に上端/下端/左端/右端の一部 row が常に黒または後段 `Pad`/`Equal` で吸収されるなら、`Concat` 入力と `gray_indices` の対応 slice を削って相対 8% 前後の改善が見込める。

## 手法

1. `gray_indices` の row `0..10` を 1 つずつ削る。
2. `Concat(axis=2) -> gray_updates` の対応入力を 1 つ削り、`gray_indices` を同じ row 数へ縮小する。
3. `src.evaluate.scorer.audit_one` で `task390.onnx` のみ検証する。
4. `n_fail=0` かつ cost 減なら submit、Public Score が上がれば採用する。
