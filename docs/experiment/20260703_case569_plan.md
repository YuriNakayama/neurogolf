## Hypothesis

The previous two-agent search was too narrow for the requested cadence. Increase
parallelism to six active search lines:

1. task102 low-fail repair
2. task343 guard-prune repair
3. task279 two-fail high-gain repair
4. task118 one-fail repair
5. broad safe candidate mining from existing scratch pools
6. micro-backlog bundle verification

The cadence target is 72 submits in 13 hours, but only submit-ready candidates
should consume submissions. A valid submit candidate must pass the local gate:
`n_fail=0`, lower cost, functions `0`, forbidden ops `[]`, and full-bundle
dry-run success. Known Kaggle `ERROR`/hidden-regression families remain excluded
unless isolated and explicitly revalidated.

## Method

1. Keep the main process free for integration, gating, and submission.
2. Run six parallel worker lines, each writing scratch candidates under
   `/private/tmp/neurogolf_case569_*`.
3. Prefer relative gain `ln(old_cost / new_cost)` and low `n_fail`, not absolute
   cost.
4. Submit immediately when a worker returns `SUBMIT_READY` and the main process
   verifies it with single-task gate plus full dry-run.
5. If no `SUBMIT_READY` candidate appears, record no-submit with diagnostics and
   start a fresh target-generation cycle instead of burning a submission.
