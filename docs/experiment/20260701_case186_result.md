## Probes

Attempted to replace:

```text
row_code_u8 = Where(Cast(row_mask_u8), sp_u8, zero_u8)
col_code_u8 = Where(Cast(col_mask_u8), sp_u8, zero_u8)
```

with:

```text
row_code_u8 = Mul(row_mask_u8, sp_u8)
col_code_u8 = Mul(col_mask_u8, sp_u8)
```

Temporary ONNX results:

- row only: invalid graph, uint8 input to `Mul` rejected by ONNXRuntime
- col only: invalid graph, uint8 input to `Mul` rejected by ONNXRuntime
- both: invalid graph, uint8 input to `Mul` rejected by ONNXRuntime

Representative error:

```text
Type Error: Type 'tensor(uint8)' of input parameter (...) of operator (Mul)
is invalid.
```

## Analysis

The rewrite is algebraically plausible for 0/1 masks, but `Mul` is not available
for uint8 tensors in this runtime/opset combination. Adding casts around the
operation would reintroduce intermediate tensors and remove the expected memory
gain.

## Conclusion

No exact cost-reducing valid ONNX was found. The baseline graph is unchanged and
no Kaggle submission was made.

## Next

Prefer `Sub` rewrites on uint8 tensors, since case183 and case185 showed they
are accepted by the runtime. Avoid uint8 `Mul` unless an existing compatible
cast can be reused without adding memory.
