## 確定（uint8-redo 仮説の精査結果）
- float-heavy floor の MaxPool は **単純 dilation でなく複雑操作**（pattern-match / CC伝播 / 多段）。
  「float MaxPool を uint8 に置換するだけ」では勝てない。規則自体が複雑で、正答再現に floor 同等の
  op 数が要る。
- uint8 が**単純に効く**のは enclosed-fill 等の純 dilation flood-fill のみ（t002/t251）で、それらは
  floor が既に uint8 適用済み（case49-50 で parity 実証）。
- ∴ 「float MaxPool floor ∩ 単純規則」の積は薄く、複雑規則タスクでは uint8 swap が純増にならない。
