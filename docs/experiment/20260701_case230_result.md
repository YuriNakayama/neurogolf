## 検証内容
`row_offsets_full_i32 (1,5,2,21)` は 5 channel に同じ offset table を繰り返していた。相対削減率を狙って table shape と update mask を縮小した。

## 結論
`ScatterElements` は `indices` と `updates` の shape を実質的に一致させる必要があり、offset table の channel broadcast は正答性を保てなかった。`update_channel_mask` も channel ごとの配置が必要で、片側化や固定化では全ペア不一致になった。

`n_fail=0` かつ cost 改善の候補がないため、ONNX は変更せず submit しない。

## 次の一手
Scatter 系の shape broadcast 削減は不発が続いている。次は Scatter ではなく、one-node `Conv`/`Einsum` の weight 表現や、bool/uint8 中間の redundant branch を持つ未検証 task を優先する。
