# 20260701 case269 plan - task300 selector OneHot rank reduction

## 背景

- Current floor: Public Score `7181.65`
- 優先基準: `n_fail=0` を絶対条件にし、絶対 cost ではなく `old_cost / new_cost` の相対削減率を優先する。
- `task300` baseline: `cost=540` (`params=66`, `memory=474`, `n_fail=0`)

`task300` は winner channel を選んだ後、以下の 4D selector を作って `Einsum` に渡している。

```text
winner_channel -> Cast(float16) -> winner_f16
winner_f16 == channel_ids_f16 -> selector
selector -> Cast(float) -> selector_f  # [1,10,1,1]
row_code = Einsum(input, selector_f, pos_weights)
col_code = Einsum(input, selector_f, pos_weights)
```

`selector_f` の spatial 次元は `1,1` で、`Einsum` 内では実質的に channel selector としてだけ使われる。

## 対象

- `data/output/onnx/task300.onnx`

## 仮説

`winner_channel` から `OneHot(depth=10, values=[0,1])` で `[1,10]` の float selector を直接作り、
2 つの `Einsum` equation を以下に変えられる。

```text
bchw,bc,h->b
bchw,bc,w->b
```

これで `winner_f16`、`selector`、`selector_f`、`channel_ids_f16` を削り、代わりに
`selector_f` 相当の `[1,10]` float `OneHot` だけを残す。削減量は小さいが baseline cost が 540 なので、
数十 bytes の削減でも `old_cost / new_cost >= 1.01` を満たす可能性がある。

## 手順

1. `/tmp` に `task300` の selector OneHot 版を生成する。
2. `src.evaluate.scorer.audit_one` で `task300` の完全一致と cost を確認する。
3. `n_fail=0` かつ `old_cost / new_cost >= 1.01` なら本体へ反映して submit する。閾値未満なら submit しない。
