# 20260701 case258 plan - task275 color Conv to ArgMax

## 背景

- Current floor: Public Score `7181.61`
- `task275` baseline: `cost=2011` (`params=637`, `memory=1374`, `n_fail=0`)
- スコア改善は `ln(old_cost/new_cost)` なので、小〜中 cost でも相対削減率が高い候補を優先する。

## 対象

- `data/output/onnx/task275.onnx`

## 仮説

`task275` の `tl_code_f` / `second_code_f` は、4色 one-hot slice に `color_kernel=[1,2,3,4]` の 1x1 Conv を当てて色番号へ戻している。

この2箇所を `ArgMax(axis=1, keepdims=1)` と `Gather([1,2,3,4])` に置換できれば、各 branch の float32 Conv output を消せる。入力 slice は残るが、2 branch 合計で約250 byte級の memory 削減が見込め、`task275` の cost では相対改善が比較的大きい。

zero cell が slice 内に存在すると Conv は 0、ArgMax は 1 になり得るため、まず guard なしの置換を `/tmp` で audit し、fail が出る場合は `ReduceMax`/`Where` guard 付きで再評価する。

## 手順

1. `/tmp` に `task275` の probe ONNX を生成し、`Conv+Cast` 2箇所を `ArgMax+Gather` に置換する。
2. `src.evaluate.scorer.audit_one` で `task275` の完全一致と cost を確認する。
3. `n_fail=0` かつ cost 減なら本体へ反映し、Kaggle submit で採否を決める。
