## 試行

`marker_color = Where(Equal(grid_u8, line_u8), zero_u8, grid_u8)` を算術化して9x9 bool maskを消す案を `/tmp` で検証した。

| 候補 | 結果 | 判定 |
|---|---|---|
| `Sub(grid_u8, line_u8)` | ORT invalid: `Sub` does not accept `tensor(uint8)` | reject |
| `Sub(line_u8, grid_u8)` | ORT invalid: `Sub` does not accept `tensor(uint8)` | reject |
| `Not(Equal) -> Cast(uint8) -> Mul(grid_u8, keep)` | ORT invalid: `Mul` does not accept `tensor(uint8)` | reject |

## 結論

opset/runtime の型制約で uint8 算術化は直接使えなかった。int32 などへ cast すれば表現できるが、9x9 の cast 中間と戻し cast が増え、今回の狙いである bool mask 1枚削減を上回るため採用しない。ONNX は変更せず submit しない。

## 次の一手

uint8 `Where` の算術化は、この runtime ではそのまま使えない。次は bool/float16 で既に算術 op が許される mask、または `Concat` 端ゼロの `Pad` 化のように型制約を受けにくい exact rewrite を優先する。
