# 20260630 case72 plan — bool→uint8 復帰 Cast の全体横展開探索

## 仮説
case71 の t349 では `uint8(0/1) -> Cast(bool) -> Cast(uint8)` の復帰 Cast が冗長で、
`Where` の値入力に元の uint8 を直結することで cost を 900 削減できた。
同じパターンが他タスクにもあれば、1 node / 1 intermediate 分の memory を削れる。

## 対象
- `data/output/onnx` 全400タスク
- 探索条件: `Cast(to=BOOL)` の出力を `Cast(to=UINT8)` に戻し、その uint8 出力を後段が使う箇所

## 手法
1. ONNX graph を走査し、`Cast(bool) -> Cast(uint8)` パターンを列挙。
2. 各候補で uint8 cast の使用先を元テンソルへ置換する probe を生成。
3. `audit_one(run_correctness=True)` で faithful 検証し、cost 減かつ n_fail=0 のものだけ採用。
