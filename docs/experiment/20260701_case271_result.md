```text
ShapeInferenceError: (op_type:Pad, node name: top0_b): data typestr: T, has unsupported type: tensor(bool)
ONNXRuntimeError: Type 'tensor(bool)' of input parameter (...) of operator (Pad) is invalid.
```

Raising opset was also not a safe fix. `opset=12` still rejects bool `Pad`, while later opsets reinterpret existing `Unsqueeze` nodes as two-input form and break shape inference.

## Conclusion

`task157` has the right relative-reduction shape, but its current opset 11 graph cannot use bool `Pad`. Converting bool masks through uint8 would add same-sized cast/pad/cast tensors and erase the savings, so the ONNX baseline is unchanged.

## Next Step

Avoid bool `Concat -> Pad` unless the task already uses an opset/runtime combination that accepts bool `Pad`. Continue with:

- numeric zero `Concat -> Pad` only when net params decrease after shared initializers are considered;
- final dtype reduction where the final computation is detached from `input`;
- memory-producing branches with `old_cost / new_cost >= 1.01`.
