## 仮説

`task118` は 2/3 のどちらかの fill kernel を選ぶ gate を持つ。arc-gen 上で片側 branch が不要、または `gate3` 比較の係数が過剰なら、branch 固定や support score 簡略化で `Einsum` / `ConvTranspose` 入力周辺の memory を減らせる可能性がある。

## 手法

1. `gate3` を constant 0 / constant 1 に固定し、未使用 branch を prune した候補を作る。
2. `support2_score` / `support3_score` の `*3` 重みを `*2` / `*1` に変えて、比較 margin が過剰か確認する。
3. `src.evaluate.scorer.audit_one` で `task118` の全ペア完全一致を確認する。
4. `n_fail=0` かつ cost が下がった候補のみ実 ONNX に反映して submit する。

## 足切り

- 禁止 op は追加しない。
- 静的 shape を維持する。
- 変更対象は `task118.onnx` のみ。
- submit は `audit_one` で `n_fail=0` かつ cost 減少の場合だけ行う。
