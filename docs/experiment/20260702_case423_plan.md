# 20260702 case423 plan - memory-heavy intermediate inspection

## Context

Case422 exhausted simple duplicate/unused constant cleanup on the current
accepted graph. Remaining improvements likely need task-local simplification of
large intermediate tensors.

Because the metric is:

```text
cost = params + memory_bytes
gain = ln(old_cost / new_cost)
```

memory-heavy tasks can clear the submit threshold if a large float tensor can be
removed, narrowed, or kept smaller without changing selector/table coverage.

## Hypothesis

Some current tasks still have one or two dominant intermediate tensors. A
task-local dtype or shape rewrite on those tensors may produce a larger
relative gain than further parameter cleanup.

## Method

1. Audit/profiling inspect current ONNX files to identify memory-heavy tasks and
   dominant tensor outputs.
2. Prioritize tasks not recently exhausted in cases417-422.
3. For one concrete task, inspect whether the dominant tensor is:
   - boolean/integer-valued but stored as float;
   - larger than the consumed spatial window;
   - a temporary that can be bypassed by an existing downstream tensor.
4. Build at most one candidate and run `submit gate`.

## Gate

Submit only if the candidate is:

```text
decision=submit-candidate
n_fail=0
gain >= 0.020
functions=0
forbidden ops=[]
```

Otherwise record no-submit and bank exact low-gain material.
