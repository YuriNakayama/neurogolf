## 背景
case79 では `TopK(k=5)` が必要で、出力構築を `Where -> Equal` に替える案は n_fail 0 だが cost 悪化だった。

現行の終端は以下:

```text
Pad fill24 -> fill
valid = valid_rows & valid_cols
out0 = valid xor fill
zero_b = fill & out0
Concat(out0, zero_b, zero_b, fill, zero_b, ..., zero_b) -> output
```

`zero_b` は論理上ほぼ false channel で、900B の dynamic intermediate になっている。これを削れれば case100 型の memory 削減に近い。

## 仮説
1. `zero_b` を static all-false initializer に置換して `And fill/out0 -> zero_b` を削除できる可能性がある。
2. `fill_4` など後段候補 branch を削ると大きく cost が落ちるが、case79 の TopK=5 必要性から正答性は厳しい。念のため最終候補のみ一点確認する。

## 手法
1. `zero_b` static 化候補を作り、`audit_one` で cost/n_fail を確認する。
2. `fill_4` branch drop 候補を作り、`audit_one` で n_fail を確認する。
3. `n_fail=0` かつ cost 減なら実ファイルに適用して Kaggle submit。

- 全数 `audit_dir` は使わず、`audit_one` のみ行う。
