## independent_impl（必須欄）
case64 の「ConvInteger 出力展開で 10ch 中間回避」技法を横展開:
- floor の **code+ConvInteger 出力**を精読: `mask10`(単ch uint8) → `code=mask+mask`(0/2) →
  `ConvInteger(code, weights[10,1,1,1] int8, zp=1)` → **output(graph出力,memory除外)に直接10ch展開**。
  10ch 中間を一切作らない。これが floor が私(10ch Concat)より安い核心。
- **横展開先探索**: 「大 float 多channel 中間あり & ConvInteger 未使用」の floor 5件
  (t044/029/174/177/310)を抽出。
- **t177 精査**（cost3822, 16node）: RoiAlign+ArgMax で **8-矩形を抽出+幾何変換**（可変出力shape）。
  大 float 中間は RoiAlign/Conv の**実計算用**で、単純 one-hot 展開でない → ConvInteger 出力技法は非適用。
  可変出力 shape で静的 net も困難。

## 確定
- **ConvInteger 出力展開は「単channel mask → one-hot 10ch」の最終展開にのみ有効**。
  大 float 多channel 中間を持つ floor は、その中間を**実計算**（RoiAlign/Conv分類）に使っており、
  単純展開で置換できない。floor は one-hot 展開が必要な箇所では既に ConvInteger 適用済み(t356)。
- ∴ ConvInteger 技法の純増余地（floor が one-hot 展開を Concat でやっている安価タスク）は見当たらず。
