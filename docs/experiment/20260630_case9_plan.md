# 20260630 case9 plan — 未解決(0点)タスクの探索 → ソルバ追加で大幅加点を狙う

## 仮説
cost golf は floor がフロンティアで余地ほぼゼロ（cycle1-8 で確定, 最大 +0.03/task）。一方
**未解決タスク 1 つにソルバを追加すれば +最大25点**で桁違いに費用対効果が高い。floor の全 400 を
faithful 監査（train/test/arc-gen 全 example, subprocess 隔離）し、n_fail>0 or status≠ok の
タスクを洗い出す。

## 対象 / 手法
- 全 400 faithful 監査 → 失敗タスクを特定。
- 失敗があれば、そのタスクの規則を numpy で厳密特定 → 最小 ONNX 化 → faithful n_fail=0 →
  combined に追加 → submit。

## 採否
- 失敗タスクにソルバ追加し faithful 通過 → submit、実 LB 上昇で採用。
- 失敗ゼロなら floor は全解決済みで、加点はソルバ追加では不可能と確定（golf のみが残る）。

## 注意
- ローカル監査は LB と完全一致しないが、arc-gen 失敗は Kaggle arc-gen でも失敗するため、
  ローカルで n_fail>0（特に arc-gen）のタスクは真に未解決の可能性が高い。
