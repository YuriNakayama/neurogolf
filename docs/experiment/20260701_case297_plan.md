## Hypothesis

The p7 branch currently computes two `7x7` MaxPool outputs and slices the first
cell from each.  Only `p7_inv_wide` has the Slice as its sole consumer:

```text
MaxPool(inv30_f, pads=[1,1,4,4], strides=[1,1]) -> p7_inv_wide [1,1,7,7]
Slice(... starts=[0,0], ends=[1,1], axes=[2,3]) -> p7_inv_cell [1,1,1,1]
```

Because the Slice keeps the first spatial cell, changing the MaxPool strides to
a value larger than the output span makes the MaxPool emit only that first cell
without changing the pooling window.  Route the `p7_inv_wide` MaxPool output
directly to `p7_inv_cell`, then remove the Slice node.

Expected effect:

- remove one `7x7` float intermediate and one Slice node;
- keep the same `1x1` cell tensors consumed downstream;
- no parameter increase;
- estimated cost reduction about `192`.

This is smaller than the desired large-delta class, but it is a direct
meaning-preserving crop of counted intermediates while larger proof-grade
candidates remain scarce.
