# 20260701 case301 plan - large-ratio candidate selection

## Context

- Current Public Score floor: `7183.04`.
- Recent accepted cycles after case291 mostly produced small local deltas:
  - case296 `task158`: `35327 -> 33717`, estimated `+0.0466`.
  - case297 `task110`: `13787 -> 13591`, estimated `+0.0143`.
  - case298 `task361`: `4839 -> 4787`, estimated `+0.0108`.
  - case300 `task080`: `16836 -> 16734`, estimated `+0.0061`.
- The scoring rule is logarithmic:

```text
points = max(1, 25 - ln(max(1, cost)))
cost   = params + memory_bytes
delta  = ln(old_cost / new_cost)
```

Therefore absolute cost reduction is not the right priority signal.  A `10%`
reduction has the same point value at any cost scale, while a fixed `-100`
cost reduction is much more valuable on a small task than on a large task.

## Updated Priority Rule

Do not prioritize tasks just because their absolute cost is large.  Prioritize
task-local changes with a large expected ratio:

```text
ratio = old_cost / new_cost
gain  = ln(ratio)
```

Selection thresholds for future submit cycles:

- hard gate: `n_fail=0`;
- primary target: `ratio >= 1.05` when a proof-grade structural rewrite exists;
- acceptable target: `ratio >= 1.02` for same-task bundled rewrites;
- avoid standalone cleanup or duplicate-initializer edits below `ratio < 1.01`
  unless no other candidate exists and the change has a direct semantic proof.

## Candidate Families

High priority:

- source-crop rewrites that remove multiple wide tensors in one task, such as
  `Conv/MaxPool -> Cast -> Slice` becoming a direct cropped producer;
- terminal dtype/final-op reductions like the earlier `task055` float16 win;
- zero-channel materialization removal only when channel layout lets `Pad` or a
  smaller terminal op preserve output without adding comparable memory;
- same-task alias bundles only when they remove whole intermediate tensors and
  the semantic equivalence is proven, not just locally observed.

Low priority:

- params-only trims below tens of elements;
- duplicate initializer cleanup by itself;
- local equality aliases without a task-level semantic proof;
- crop-before-cast rewrites when the pre-cast dtype is wider, because they often
  add a larger cropped intermediate than the bool/uint8 tensor they remove.
