## 仮説
入力が one-hot なら `Conv(cw)` は `ArgMax(axis=1, keepdims=1)` と等価に色番号を返せる。`ArgMax` 後に uint8 へ Cast すれば、float32 `code_f` と後続 Cast を削れて memory を大きく減らせる可能性がある。

## 検証
- `Conv(cw)` を `ArgMax + Cast(uint8)` に置換し、`rowcolor/colcolor/iszero` 周辺を uint8 入力へ差し替えて `audit_one` する。
- `rowcolor_f` / `colcolor_f` の Cast 削除 alias を probe する。
- 終端 `Where` (`isvert`, `mark`, `ingrid`) の固定化も低コストで確認する。
- `src.evaluate.scorer.audit_one` で `task202` の全ペア完全一致と cost を確認し、`n_fail=0` かつ十分な cost 改善候補だけ実 ONNX へ反映する。
