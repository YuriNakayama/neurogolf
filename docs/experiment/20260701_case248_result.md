## 試行
row/col の last endpoint 算出を、座標 `MatMul` から `ArgMax(select_last_index=1)` に置換した。

### 1. guard なし

```text
cost=3969
n_pass=0
n_fail=271
```

空行・空列で `ArgMax(select_last_index=1)` が最後のゼロ位置を返すため、全例で出力が崩れた。

### 2. presence guard あり

空行・空列を 0 に戻すため、`ReduceMax(x8)` の presence を int8 に cast し、last index に `Mul` した。

```text
params=57
memory=4125
cost=4182
n_pass=271
n_fail=0
```

完全一致にはなったが、`MatMul` の中間は 15 要素程度と小さく、追加した `ArgMax` / `ReduceMax` / `Cast` / `Mul` の memory が上回った。

## 結論
不採用。

`task050` の endpoint 算出は現行 MatMul 版の方が scorer 上安い。ONNX 本体は変更しない。
