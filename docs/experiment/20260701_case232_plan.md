# case232 plan: task101 false work tensor trim

## 現状
- 採用済み Public Score floor: 7181.21
- `task101` baseline は `cost=17755` (`params=905`, `memory=16850`, `n_fail=0`)。
- graph は 341 nodes で、`Where`/`Cast`/`And`/`Add` が多い memory 支配タスク。
- `false_work_flat` は shape `(357,)` の all-false bool initializer で、終盤の `ScatterElements(false_work_flat, blue_safe32, blue_active)` の base data に使われる。
- `offwork_f` は shape `(357,)` の float16 descending table で、位置 score に使われる。

## 対象
- `data/output/onnx/task101.onnx`

## 仮説
`false_work_flat` は全 false なので scalar や小 shape に縮小できれば params と memory を削れる可能性がある。`ScatterElements` が data broadcast を許さない場合は失敗する。`offwork_f` は Range/Neg 生成も可能だが、追加中間 memory で悪化する可能性が高いため、まず false base tensor に絞る。

## 検証
- `false_work_flat` を scalar / `(1,)` / smaller shape に縮小して `audit_one` する。
- `ScatterElements` base を既存 `zero_bool` に alias できるか probe する。
- `offwork_f` の dtype/shape 変更が可能か最小限 probe する。
- `src.evaluate.scorer.audit_one` で `task101` の全ペア完全一致と cost を確認し、`n_fail=0` かつ cost が下がった候補だけ実 ONNX へ反映する。
