# case245 result: task160 constant feature bias fold

## 結果
- 対象: `data/output/onnx/task160.onnx`
- baseline: `cost=1334` (`params=170`, `memory=1164`, `n_pass=265`, `n_fail=0`)
- 採否: 不採用
- Kaggle submit: なし

## 検証内容
`features_u8 = Concat(fg_u8, square_u8, plus_u8, ones_u8)` の `ones_u8` channel を、終端 `QLinearConv` の bias に移す候補を probe した。

| 候補 | cost | params | memory | n_pass | n_fail | 判定 |
|---|---:|---:|---:|---:|---:|---|
| baseline | 1334 | 170 | 1164 | 265 | 0 | ok |
| `ones_u8` channel -> `final_bias` | 1134 | 70 | 1064 | 0 | 265 | reject |

## 結論
単純な bias fold は不正解。終端 `QLinearConv` は `pads=[0,0,20,20]` で 10x10 入力を 30x30 output に直接広げている。元の `ones_u8` channel の寄与は 10x10 入力範囲だけだが、bias は 30x30 output 全域に掛かるため、pad 領域の値が変わる。

faithful にするには、bias fold した 10x10 conv output を別途 `Pad` で 30x30 に戻す構成が必要だが、10x10x10 の中間 tensor が追加され、削減できる params/memory より memory 増が大きい見込み。`n_fail=0` かつ cost 改善候補がないため、ONNX は変更せず submit しない。

## 次の一手
constant feature の bias fold は、終端 Conv が pad なしで同じ空間だけを出力している場合に限定して狙う。pad 付き Conv では bias が pad 領域にも入るため、mask/Padding 補正の cost を事前に見積もる。
