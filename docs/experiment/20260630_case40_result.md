## 確定
- **高コストタスクは「規則が本質的に非局所・逐次だから高コスト」**:
  flood-fill(t367/t187/t364/t002) / path-tracing(t286) / connected-components / 多段変換(t233/t018)。
- 手作業構築は規則を表す op 数が原理的に多く、memory=Σ中間 で floor の最適化済み net を超える。
  floor は各高コストタスクで bit-parallel / quantized / bool 中間を駆使し near-optimal。
- ノード数が少ない高コスト task(t367=11)も、中身は flood-fill で逐次反復が必須 → 手構築不可。

## 構造的結論（再確認）
cost = params + Σ(全中間テンソル)。floor は各タスクで「単一 scorable op(中間ゼロ) or
bool/quant 中間の最小多ノード」に到達済み。独自実装(手構築/学習fit/合成/sparse)はいずれも
中間テンソルを足す分だけ不利で、現プリミティブ・cost モデル下で floor を下回れない。
これは case30/35/36/37/38/39/40 で計7サイクル・実 net 構築/計測/実Kaggle検証により実証。
