## independent_impl（必須欄・新規アルゴリズム考案）
t204(floor 11494, enclosed-fill+サイズ分類)を手構築で攻略:
- **規則を厳密 decode**: 囲み正方形を **side parity** で色分け（even side→2, odd side→7）。268/268。
- **CC-free 大域アルゴリズムを考案**（NonZero/CC 不要、許可op のみ）:
  1. border から flood → interior(enclosed)。
  2. 反復 erosion で **isolated singleton**(=odd 正方形の中心は最後に1×1で残る) を seed 検出。
  3. seed を interior 上に dilate 伝播 → odd 領域=7, 残り=2。
  - **numpy で 268/268 完全一致**。MaxPool/erosion(MinPool=1-MaxPool(1-x))/Mul/Sub のみ＝uint8 可。
- **cost 評価（決定的）**: 3相反復（flood ~10 + erosion 6×5op + propagate ~10）≈ **80 中間**。
  20×20 uint8 でも **~32000 ≫ floor 11494**。ONNX 構築は topological 順序の bug 残（logic は numpy で実証済）。

## 確定
- t204 の規則は**CC-free で厳密に解けた**（新規 erosion-seed-propagate アルゴリズム, 268/268）。
  これは「解けない」のではない — 独自アルゴリズムで正答構成を発見した。
- だが **分類付き enclosed-fill は flood+erosion+propagate の3相反復が必須**で、各相が中間を生み、
  uint8 でも floor 11494 を超える。floor は同分類をより少 op で実装（おそらく CC を直接 or 別表現）。
- 手構築(本case57)で cost 不可を実証。学習fit も同規則を CNN で学べるが3相相当の depth で同様に不利。

## 知見（分類付きタスクの一般性）
- enclosed-fill + 分類(サイズ/形状) は本質的に **多相反復**（fill→測定→再着色）で、cost が嵩む。
  純粋 fill(t002/251)が parity 止まりなのに対し、分類付き(t187/t204)は floor 比で更に不利。
- 独自実装で「規則は解ける」が「floor の cost に届かない」構造が分類タスクで顕著。
