## Context

Accepted Public Score remains `7186.71`. After many no-submit re-gates, this
cycle returns to relative-gain ranking from the current ONNX graphs.

`task371` is a very small task, but its cost is dominated by one index tensor:


Graph:


Dominant tensors / initializers:


Because the score is logarithmic, saving even 160-320 bytes on a `cost=467`
task would be a large relative gain. The likely blocker is runtime support:
previous `ScatterND` probes on other tasks rejected int32 indices.

## Hypothesis

If `ScatterND` accepts int32 indices in this graph, converting the index path
from int64 to int32 would be a strong low-risk candidate. If it rejects int32,
the task should be marked no-submit because adding casts would reintroduce the
same large tensor.

## Method

1. Re-gate existing `task371` cleanup/index/dtype scratch candidates.
2. Build a bounded int32 index-path candidate:
   - `zero_index` int64 -> int32;
   - `scatter_offsets` int64 -> int32;
   - casts to `row_center_i` / `col_center_i` target int32;
   - remove stale value_info and infer shapes.
3. Run the submit gate against only `task371`.
4. Submit only if the candidate is exact, runtime-valid, and clears the gate.
