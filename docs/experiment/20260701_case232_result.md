## 検証内容
`false_work_flat (357,)` は all-false bool initializer で、終盤の `ScatterElements` の base data に使われる。相対削減率を狙って scalar/small shape 化、既存 `zero_bool` alias、`offwork_f` の表現変更を probe した。

| 候補 | cost | n_pass | n_fail | 判定 |
|---|---:|---:|---:|---|
| baseline | 17755 | 266 | 0 | ok |
| `false_work_flat -> scalar false` | 17043 | 0 | 266 | reject |
| `false_work_flat -> (1,)` | 17043 | 0 | 266 | reject |
| `false_work_flat -> (90,)` | 17221 | 0 | 266 | reject |
| `false_work_flat -> (360,)` | 17761 | 0 | 266 | reject |
| `false_work_flat -> uint8(357)` | invalid | 0 | 0 | reject: `ScatterElements` dtype mismatch |
| `false_work_flat -> zero_bool` alias | 17399 | 0 | 266 | reject |
| `offwork_f -> int16/float32` | invalid | 0 | 0 | reject: `Where` dtype mismatch |
| `offwork_f -> (356,)` | invalid | 0 | 0 | reject: shape mismatch |
| `offwork_f -> abs` | 17755 | 9 | 257 | reject |
| `offwork_f` via float16 `Range` | invalid | 0 | 0 | reject: `Range` float16 invalid |

## 結論
`false_work_flat` は all-false だが、`ScatterElements` の data tensor shape として必要だった。shape を縮めると cost は下がるものの全ペア不一致になる。`offwork_f` は float16 同 shape が必要で、dtype 変更や Range 生成は ONNX Runtime の型制約に合わない。

`n_fail=0` かつ cost 改善の候補がないため、ONNX は変更せず submit しない。

## 次の一手
all-zero/all-false base tensor の shape 縮小は Scatter 系で不発が続く。次は Scatter を避け、`Where(Cast(mask), value, zero)` が複数ある中 cost task で arithmetic 化や mask 共有の exact 変形を探す。
