## 結論

この runtime の `ScatterND` は int32 indices を受け付けず、int64 が必要だった。最後に int64 cast を戻す案は int32 concat と int64 cast output の両方が残って memory が増えるため採用しない。ONNX は変更せず submit しない。

## 次の一手

Scatter 系の index dtype 削減は `ScatterElements` など実際に int32 を受ける op に限定して試す。`ScatterND` では dtype 削減より、updates や terminal mask の exact rewrite を優先する。
