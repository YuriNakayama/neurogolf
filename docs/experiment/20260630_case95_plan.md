# 20260630 case95 plan — t243 の 28段 flood 反復短縮を検証する

## 背景

- 現 Public best は **7180.87**。
- `task243.onnx` は faithful baseline で cost **22650**（params 42, memory 22608）。
- Graph は `ch1` を seed にして `QLinearConv(CROSS_W) -> Where(is_zero)` を 28段展開し、最後に `fill_27` を pad して出力する。
- 過去ログでは真の規則は 2D flood とされ、単純 row-separable solver は棄却済み。

## 仮説

28段の最終数段が arc-gen 上余剰なら、`fill_27` を `fill_26` / `fill_25` などへ置換して 1段あたり 2 tensor 程度を削れる。case84 のような余剰反復が残っている可能性を確認する。

## 対象タスク

- `task243`

## 手法

1. ORT profile で memory 上位 tensor を確認する。
2. `/tmp` variant で final `Cast(fill_27)` の入力を `fill_26`, `fill_25`, `fill_24`, `fill_23` へ置換し、prune 後に `audit_one`。
3. さらに初期 `one_cross -> fill0` を bypass する短縮候補も試す。
4. faithful `n_fail=0` かつ cost 減なら submit する。

## 採否

- faithful `audit_one` が `n_fail=0` かつ cost が 22650 未満なら `task243.onnx` を差し替えて submit する。
- Public Score が 7180.87 を上回れば採用し、DVC push と git commit/push を行う。
- 壊れる候補、または cost が変わらない候補は破棄する。
